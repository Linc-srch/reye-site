Highly Critical Drupal Core Flaw Exposes PostgreSQL Sites to RCE Attacks

Highly Critical Drupal Core Flaw Exposes PostgreSQL Sites to RCE Attacks

supply-chain-attack
Summary

Drupal released security updates for a highly critical vulnerability (CVE-2026-9082) affecting its database abstraction API, which can lead to SQL injection on PostgreSQL-based sites. Anonymous users can exploit this flaw to achieve remote code execution, privilege escalation, or information disclosure. The vulnerability has a CVSS score of 6.5. Fixed versions include Drupal 11.3.10, 11.2.12, 11.1.10, 10.6.9, 10.5.10, and 10.4.10. Proof-of-concept exploits have been published by Searchlight Cyber. Only sites using PostgreSQL are impacted; MySQL and SQLite installations are not vulnerable, but upgrades are recommended due to bundled security updates for Symfony and Twig.

摘要

Drupal 发布安全更新修复了一个严重漏洞 CVE-2026-9082，该漏洞位于数据库抽象 API 中，可导致针对使用 PostgreSQL 的站点进行 SQL 注入攻击，进而实现远程代码执行、权限提升或信息泄露。匿名用户可利用此漏洞，CVSS 评分为 6.5。已修复版本包括 Drupal 11.3.10、11.2.12 等。Searchlight Cyber 已发布 PoC 代码。该漏洞仅影响使用 PostgreSQL 的部署，但建议所有用户升级以获取 Symfony 和 Twig 的 bundled 安全更新。
