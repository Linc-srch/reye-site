Webworm Deploys EchoCreep and GraphWorm Backdoors Using Discord and MS Graph API

Webworm Deploys EchoCreep and GraphWorm Backdoors Using Discord and MS Graph API

启示 · Insight

For Harmony Security and SOC researchers, this report provides critical intelligence on how advanced threat actors are weaponizing trusted third-party platforms (Discord, Microsoft Graph API, GitHub) for C2 operations. This directly impacts the 'Threat Vectors, Vulnerabilities & Data Privacy' and 'Advanced AI/Agent Security Technologies & Defenses' taxonomies. The use of legitimate APIs for malicious purposes underscores the need for AI-driven anomaly detection systems that can baseline normal API traffic patterns and detect deviations, rather than relying solely on signature-based detection. For SOC teams, this highlights a blind spot in monitoring: traffic to legitimate cloud services (OneDrive, Discord) may be malicious, requiring behavioral analysis and UEBA integration. The shift from custom RATs to proxy tools also informs endpoint security strategies, emphasizing the need to monitor for unauthorized proxy software and VPN usage.

对于 Harmony Security 和 SOC 研究人员而言，这份报告提供了关键情报，揭示了高级威胁行为者如何将受信任的第三方平台（Discord、Microsoft Graph API、GitHub）武器化用于 C2 操作。这直接影响“威胁向量、漏洞与数据隐私”和“先进 AI/Agent 安全技术与防御”分类。利用合法 API 进行恶意活动，凸显了对基于 AI 的异常检测系统的需求，该系统能够建立正常 API 流量模式基线并检测偏差，而非仅依赖基于签名的检测。对于 SOC 团队，这突出了一个监控盲点：流向合法云服务（OneDrive、Discord）的流量可能是恶意的，需要行为分析和 UEBA 集成。从定制 RAT 向代理工具的转变也为端点安全策略提供了信息，强调需要监控未经授权的代理软件和 VPN 使用。

Summary

China-aligned threat actor Webworm has deployed two new custom backdoors, EchoCreep and GraphWorm, which leverage Discord and Microsoft Graph API for stealthy command-and-control communications, marking a shift from traditional RATs to semi-legitimate proxy tools.

摘要

疑似具有中国背景的威胁行为者 Webworm 部署了两种新的定制后门 EchoCreep 和 GraphWorm，它们利用 Discord 和 Microsoft Graph API 进行隐蔽的命令与控制通信，标志着其从传统 RAT 向半合法代理工具的转变。

Deep Note

Key Points

  • Webworm, a China-aligned threat actor active since at least 2022, has deployed two new backdoors: EchoCreep (uses Discord for C2) and GraphWorm (uses Microsoft Graph API for C2).
  • The group has shifted from traditional RATs (Trochilus, Gh0st, 9002) to semi-legitimate proxy tools (SoftEther VPN, iox, WormFrp, ChainWorm, SmuxProxy, WormSocket) for stealth.
  • A GitHub repository impersonating a WordPress fork is used as a staging ground for malware and tools.
  • EchoCreep supports file upload/download and command execution; GraphWorm can spawn cmd.exe, execute processes, upload/download to OneDrive, and self-terminate.
  • Over 433 Discord C2 messages were sent to more than 50 unique targets since March 2024.
  • Initial access vectors are unknown but involve open-source tools like dirsearch and nuclei for brute-forcing and vulnerability scanning.
  • Links to the Space Pirates cluster are considered tenuous by ESET, based on shared open-source RAT usage.
  • A separate disclosure covers a BadIIS variant sold under a MaaS model by an actor using the alias 'lwxat'.

Context & Contrast

This development marks a significant evolution in Webworm's TTPs. Historically, the group relied on well-known, often signatured RATs. The shift to custom backdoors that abuse legitimate APIs (Discord, Microsoft Graph) and the adoption of proxy tools represent a move toward 'living off the land' and 'trusted platform abuse' techniques. This contrasts with earlier, more overt malware deployments. The use of GitHub as a staging ground is also a notable tactic to blend with legitimate developer activity. The report also clarifies that the link to Space Pirates is weaker than previously thought, refining the threat intelligence picture.

Why It Matters

This matters because it demonstrates a sophisticated, persistent threat actor adapting to modern defenses by abusing trusted platforms that are often whitelisted or poorly monitored. The use of Discord and Microsoft Graph API for C2 makes detection significantly harder for traditional network security tools. For AI security, it highlights the need for models that can distinguish between benign and malicious API usage patterns. For SOCs, it underscores the necessity of monitoring cloud application traffic and endpoint proxy usage. The shift to proxy tools also complicates forensic analysis and attribution.

Critical Take

While the report is from a reputable source (ESET), several caveats exist. The initial access vector remains unknown, which is a critical gap in understanding the full attack chain. The claimed link to China is based on open-source intelligence and may be contested. The report does not provide specific indicators of compromise (IOCs) or detection rules for the new backdoors, limiting immediate defensive utility. The 'tenuous' link to Space Pirates suggests some earlier threat intelligence may have been overconfident. The BadIIS disclosure is a separate, albeit related, story and should not be conflated with Webworm's activities.

What to Watch

  • Publication of IOCs, YARA rules, or detection signatures for EchoCreep and GraphWorm.
  • Further analysis of the Discord C2 channel and Microsoft Graph API usage patterns.
  • Identification of the initial access vector (e.g., specific vulnerabilities or phishing campaigns).
  • Monitoring for similar abuse of other legitimate APIs (e.g., Slack, Telegram, Google Drive) by Webworm or other actors.
  • Tracking the evolution of the BadIIS MaaS offering and its adoption by other groups.
  • Any reports of Webworm targeting AI/ML infrastructure or using AI-generated content for social engineering.
深度笔记

要点

  • 至少自 2022 年以来活跃的疑似中国背景威胁行为者 Webworm 部署了两种新的后门：EchoCreep（使用 Discord 进行 C2）和 GraphWorm（使用 Microsoft Graph API 进行 C2）。
  • 该组织已从传统 RAT（Trochilus、Gh0st、9002）转向半合法代理工具（SoftEther VPN、iox、WormFrp、ChainWorm、SmuxProxy、WormSocket）以增强隐蔽性。
  • 一个冒充 WordPress 分支的 GitHub 仓库被用作恶意软件和工具的暂存地。
  • EchoCreep 支持文件上传/下载和命令执行；GraphWorm 可以生成 cmd.exe、执行进程、上传/下载到 OneDrive 并自我终止。
  • 自 2024 年 3 月以来，超过 433 条 Discord C2 消息被发送到 50 多个独特目标。
  • 初始访问向量未知，但涉及使用 dirsearch 和 nuclei 等开源工具进行暴力破解和漏洞扫描。
  • ESET 认为与 Space Pirates 集群的联系很薄弱，基于共享的开源 RAT 使用。
  • 另一份披露涉及一个由化名“lwxat”的行为者以 MaaS 模式销售的 BadIIS 变体。

背景与对比

这一发展标志着 Webworm 的 TTP 发生了显著演变。历史上，该组织依赖知名的、通常有签名的 RAT。转向利用合法 API（Discord、Microsoft Graph）的定制后门以及采用代理工具，代表了向“离地生存”和“滥用可信平台”技术的转变。这与早期更公开的恶意软件部署形成对比。使用 GitHub 作为暂存地也是一种值得注意的策略，旨在融入合法的开发者活动。该报告还澄清了与 Space Pirates 的联系比之前认为的要弱，从而完善了威胁情报图景。

重要性

这很重要，因为它展示了一个复杂、持久的威胁行为者通过滥用通常被列入白名单或监控不足的可信平台来适应现代防御。使用 Discord 和 Microsoft Graph API 进行 C2 使得传统网络安全工具的检测难度显著增加。对于 AI 安全，它凸显了需要能够区分良性 API 使用模式和恶意 API 使用模式的模型。对于 SOC，它强调了监控云应用流量和端点代理使用的必要性。向代理工具的转变也使取证分析和归因复杂化。

批判性视角

虽然该报告来自知名来源（ESET），但存在几个注意事项。初始访问向量仍然未知，这是理解完整攻击链的关键缺口。声称与中国有关联是基于开源情报，可能存在争议。该报告未提供新后门的具体入侵指标（IOC）或检测规则，限制了直接的防御效用。与 Space Pirates 的“薄弱”联系表明一些早期的威胁情报可能过于自信。BadIIS 的披露是一个独立但相关的事件，不应与 Webworm 的活动混为一谈。

后续关注

  • 发布 EchoCreep 和 GraphWorm 的 IOC、YARA 规则或检测签名。
  • 进一步分析 Discord C2 频道和 Microsoft Graph API 使用模式。
  • 识别初始访问向量（例如，特定漏洞或钓鱼活动）。
  • 监控 Webworm 或其他行为者是否类似地滥用其他合法 API（例如 Slack、Telegram、Google Drive）。
  • 追踪 BadIIS MaaS 产品的演变及其被其他组织的采用情况。
  • 任何关于 Webworm 针对 AI/ML 基础设施或使用 AI 生成内容进行社会工程的报告。
