SafeCtrl-RL: Inference-Time Adaptive Behaviour Control for LLM Dialogue via RL-Driven Prompt Optimisation

SafeCtrl-RL：基于强化学习驱动的提示优化的推理时自适应行为控制

content-management rlhf zero-trust

Michael Orme, Yanchao Yu, Zhiyuan Tan

Summary

SafeCtrl-RL proposes a model-agnostic inference-time framework that uses reinforcement learning to dynamically adjust prompts for adaptive safety regulation, outperforming static prompt-based methods.

摘要

本文提出了一种模型无关的推理时安全控制框架SafeCtrl-RL，利用强化学习动态调整提示词，无需重新训练或修改参数即可提升LLM对话的安全性。实验表明，该方法在多个LLM上优于静态提示优化方法，并在安全性与响应质量之间取得了良好平衡。

1) Why-read

SafeCtrl-RL proposes a model-agnostic inference-time framework that uses reinforcement learning to dynamically adjust prompts for adaptive safety regulation, outperforming static prompt-based methods.

2) CRGP

C — Context

Ensuring safe and contextually appropriate behavior in LLMs is critical for real-world deployment. Existing safety approaches rely on offline alignment techniques like fine-tuning and filtering, which are costly to update and lack dynamic adaptability during interaction.

R — Related work

  • Parameter-based editing methods (e.g., ROME, MEMIT) modify internal model representations but require white-box access and operate offline.
  • Prompt-based approaches (e.g., Self-Correction, OPRO, GRIPS) steer behavior through input conditioning but typically use static or offline-optimized prompts.
  • Iterative refinement methods introduce multi-step generation but employ fixed strategies without adaptive context-sensitive control.

G — Gap

Current safety methods are either costly to update (parameter-based) or lack adaptive, context-sensitive control during inference (prompt-based). There is a need for a framework that can dynamically adjust behavior at inference time without model retraining or parameter access.

P — Proposal

SafeCtrl-RL models dialogue safety as a closed-loop control problem, using an RL agent to select prompt adjustment strategies based on a 36-dimensional state vector encoding dialogue dynamics and optimization progress. The agent iteratively refines the system prompt until a safety-quality evaluator (using DeepEval and Gemini 2.0 Flash) confirms a score above 0.9.

---

4) Experiments

Main results

Evaluated across multiple LLMs and unsafe dialogue scenarios, SafeCtrl-RL consistently improves safety and response quality, outperforming handcrafted and prompt optimization baselines. It achieves favorable performance-efficiency trade-offs, with partial retention of improved behaviors after safeguard removal.

Ablation

The paper does not provide explicit ablation study numbers in the truncated body, but the framework's design includes ablation of state features (meta-learning, score, prompt) and action space strategies (11 refinement strategies).

Limitations

['The framework relies on a safety-quality evaluator (Gemini 2.0 Flash) which may introduce latency and cost.', 'The 36-dimensional state representation may not capture all nuanced dialogue contexts.', 'Partial retention of improved behaviors after safeguard removal suggests potential instability in long-term deployment.']

5) Why it matters

  1. Provides a scalable, model-agnostic approach to LLM safety that can be deployed without retraining.
  2. Demonstrates that inference-time behavioral control via RL can achieve competitive safety improvements.
  3. Opens avenues for adaptive safety in interactive AI systems where context evolves dynamically.

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：确保LLM的安全与上下文适当行为对实际部署至关重要，现有安全方法依赖离线对齐技术（如微调、过滤），更新成本高且缺乏交互中的动态适应性。 相关工作：基于参数编辑的方法（如ROME、MEMIT）需白盒访问且离线操作；基于提示的方法（如Self-Correction、OPRO、GRIPS）通过输入条件引导行为，但通常使用静态或离线优化的提示；迭代精炼方法引入多步生成，但采用固定策略，缺乏自适应上下文感知控制。 差距：现有安全方法要么更新成本高（参数型），要么在推理时缺乏自适应上下文感知控制（提示型）。需要一种无需模型重训练或参数访问即可在推理时动态调整行为的框架。 方案：SafeCtrl-RL将对话安全建模为闭环控制问题，使用RL智能体基于编码对话动态与优化进度的36维状态向量选择提示调整策略。智能体迭代精炼系统提示，直到安全质量评估器（使用DeepEval和Gemini 2.0 Flash）确认得分超过0.9。

3) 实验

在多个LLM和不安全对话场景下评估，SafeCtrl-RL持续提升安全性与响应质量，优于手工设计及提示优化基线。实现了有利的性能-效率权衡，移除安全机制后部分改进行为得以保留。
