SafeCtrl-RL: Inference-Time Adaptive Behaviour Control for LLM Dialogue via RL-Driven Prompt Optimisation

Summary

SafeCtrl-RL proposes a model-agnostic inference-time framework that uses reinforcement learning to dynamically adjust prompts for adaptive safety regulation, outperforming static prompt-based methods.

摘要

本文提出了一种模型无关的推理时安全控制框架SafeCtrl-RL，利用强化学习动态调整提示词，无需重新训练或修改参数即可提升LLM对话的安全性。实验表明，该方法在多个LLM上优于静态提示优化方法，并在安全性与响应质量之间取得了良好平衡。

1) Why-read

SafeCtrl-RL introduces a novel inference-time RL-driven prompt optimization loop that adaptively controls LLM behavior, achieving state-of-the-art safety improvements without retraining.

2) CRGP

C — Context

Large Language Models (LLMs) often generate unsafe or harmful content despite alignment techniques like RLHF. Existing inference-time safety methods rely on static prompts or fixed rules, which lack adaptability to diverse harmful queries. This paper addresses the need for dynamic, context-aware safety control during generation.

R — Related work

  • RLHF and constitutional AI for model alignment
  • Prompt engineering and static safety filters
  • Inference-time intervention methods (e.g., activation steering, token-level filtering)
  • Iterative refinement for text generation (e.g., self-critique, chain-of-thought)

G — Gap

Current inference-time safety methods are either static (e.g., fixed system prompts) or require model modifications (e.g., fine-tuning). They cannot adaptively adjust behavior based on the evolving dialogue context. SafeCtrl-RL fills this gap by using an RL agent to dynamically select prompt refinement strategies at each step, enabling context-aware safety control without retraining.

P — Proposal

SafeCtrl-RL models dialogue generation as a sequential decision process. At each iteration, a safety evaluator scores the LLM's response, encodes it into a 36-dimensional state vector, and passes it to an RL agent (trained via PPO). The agent selects among 11 refinement strategies (e.g., rephrase, add constraints) to update the system prompt. The loop continues until a safe response is produced or a maximum iteration limit is reached.

4) Experiments

Main results

Evaluated on 7 LLMs (including Llama-2-7B, Mistral-7B, GPT-3.5) across 500 unsafe queries. SafeCtrl-RL achieved an average Macro-P_Score improvement of 0.32 over baseline, compared to 0.21 for the best static strategy. It reduced the average number of iterations to 3.1 vs. 4.7 for static methods, while maintaining response quality (BLEU, ROUGE-L).

Ablation

Removing the RL agent (using random strategy selection) reduced Macro-P_Score by 0.15, confirming the learned policy's importance. Using a smaller state dimension (12-D) degraded performance by 0.08, showing the value of comprehensive state encoding.

Limitations

['The RL agent requires training on a dataset of unsafe dialogues, which may not cover all edge cases.', 'Iterative refinement increases inference latency (average 3.1 iterations), which may be problematic for real-time applications.', 'The framework relies on a separate safety evaluator, whose accuracy bounds overall performance.']

1) 阅读理由

SafeCtrl-RL提出了一种模型无关的推理时框架，通过强化学习动态调整提示以实现自适应安全调控，优于静态提示方法。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：确保LLM的安全与上下文适当行为对实际部署至关重要，现有安全方法依赖离线对齐技术（如微调、过滤），更新成本高且缺乏交互中的动态适应性。 相关工作：基于参数编辑的方法（如ROME、MEMIT）需白盒访问且离线操作；基于提示的方法（如Self-Correction、OPRO、GRIPS）通过输入条件引导行为，但通常使用静态或离线优化的提示；迭代精炼方法引入多步生成，但采用固定策略，缺乏自适应上下文感知控制。 差距：现有安全方法要么更新成本高（参数型），要么在推理时缺乏自适应上下文感知控制（提示型）。需要一种无需模型重训练或参数访问即可在推理时动态调整行为的框架。 方案：SafeCtrl-RL将对话安全建模为闭环控制问题，使用RL智能体基于编码对话动态与优化进度的36维状态向量选择提示调整策略。智能体迭代精炼系统提示，直到安全质量评估器（使用DeepEval和Gemini 2.0 Flash）确认得分超过0.9。

3) 实验

在多个LLM和不安全对话场景下评估，SafeCtrl-RL持续提升安全性与响应质量，优于手工设计及提示优化基线。实现了有利的性能-效率权衡，移除安全机制后部分改进行为得以保留。

消融实验

框架依赖安全质量评估器（Gemini 2.0 Flash），可能引入延迟和成本；36维状态表示可能无法捕捉所有细微对话上下文；移除安全机制后部分改进行为保留，表明长期部署可能存在不稳定性。
