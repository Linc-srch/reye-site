1) Why-read

LLM-guided tree search autonomously generates and optimizes infectious disease forecasting models that match or beat human-curated CDC ensembles in real-time.

2) CRGP

C — Context

Probabilistic forecasting of infectious diseases is critical for public health but relies on labor-intensive manual model curation by expert teams. The CDC runs collaborative forecasting hubs (FluSight, COVIDHub, RSVHub) where diverse models are submitted weekly. Prior work showed LLM-based tree search (ERA) could retrospectively match expert models for COVID-19.

R — Related work

  • CDC collaborative forecasting hubs (FluSight, COVIDHub, RSVHub) with weekly submissions and ensemble aggregation
  • Prior ERA work (arXiv: 2405.12345) showing retrospective COVID-19 forecasting with LLM-guided tree search
  • AlphaGo-style Monte Carlo Tree Search applied to code generation for scientific tasks
  • Traditional mechanistic (compartmental), statistical (ARIMA), and ML (random forest, neural network) disease models

G — Gap

Retrospective evaluations risk data leakage and fail to capture real-time challenges like reporting delays, holiday anomalies, and data backfill. No prior work demonstrated fully autonomous, prospective multi-pathogen forecasting that competes with human-curated ensembles across multiple diseases simultaneously.

P — Proposal

The system uses LLM-guided Monte Carlo Tree Search (ERA) to iteratively generate, evaluate, and refine Python code for forecasting models. It seeds the search with prompts spanning established methods, recombinations, and novel approaches, then selects models based on combined validation and test performance. An automated judge-in-the-loop ensures structural fidelity to scientific theories, and log-scale distance metrics prevent reward hacking.

3) Figures

4) Experiments

Main results

In the 2025-2026 US respiratory season, ERA generated 142 unique prompts, 207,500+ candidate models, and selected 54 for an internal hub (19 for ensemble). The ERA ensemble matched or outperformed the CDC hub ensembles for influenza, COVID-19, and RSV. For RSV, it successfully handled data-scarce 'cold start' scenarios.

Ablation

Optimizing log-scale distance metrics (e.g., log-score) instead of linear metrics prevented reward hacking (overfitting to validation noise). Adding an LLM-as-judge fidelity gate improved structural adherence to scientific theory, reducing nonsensical model variants.

Limitations

The system still requires expert-crafted prompts and task specifications. Computational cost of tree search is high (207k+ models evaluated). Performance may degrade for pathogens with very limited historical data or during unprecedented pandemic dynamics.

5) Why it matters

  1. Demonstrates LLMs can automate expert-level model discovery, reducing human labor bottleneck for scaling forecasting to granular geographies or emerging pathogens.
  2. Prospective validation with auditable timestamps sets a new standard for evaluating AI-driven scientific discovery in real-world settings.
  3. The framework's ability to hybridize diverse methodologies (mechanistic, statistical, ML) autonomously could generalize beyond epidemiology to other time-series forecasting domains.

6) Next steps

  • [ ] Test ERA on forecasting tasks for vector-borne diseases (e.g., dengue, malaria) with different data structures.
  • [ ] Reduce computational cost via smarter search pruning or cheaper surrogate models for candidate evaluation.
  • [ ] Explore fully autonomous prompt generation from raw epidemiological literature without human-crafted seeds.
  • [ ] Integrate real-time data streams (e.g., wastewater, mobility) to improve cold-start and emerging pathogen performance.
