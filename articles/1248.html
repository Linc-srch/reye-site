CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

supply-chain-attack
启示 · Insight

For SOC & AI Security: CVE-2025-34291 in Langflow (an AI/LLM orchestration tool) demonstrates a critical attack surface where AI infrastructure (CORS misconfig, no CSRF, code-exec endpoint) is exploited for initial access, leading to cascading compromise of downstream cloud/SaaS APIs. This directly informs monitoring requirements for AI agent deployments and edge-cloud environments. For System Security: CVE-2026-34926 highlights a classic directory traversal in an on-premise endpoint protection product (Apex One), requiring pre-authenticated admin access to the server, underscoring that even security software can be a vector for lateral movement and code injection once initial foothold is gained.

对于 SOC 与 AI 安全：Langflow（AI/LLM 编排工具）中的 CVE-2025-34291 展示了一个关键攻击面——AI 基础设施（CORS 配置错误、无 CSRF 防护、代码执行端点）被用于初始访问，导致下游云/SaaS API 的级联沦陷。这直接为 AI 智能体部署和端云环境的监控需求提供了依据。对于系统安全：CVE-2026-34926 凸显了传统端点防护产品（Apex One）中的目录遍历漏洞，需要攻击者已获取服务器管理员凭据，表明即使是安全软件本身，在攻击者获得初始立足点后也可能成为横向移动和代码注入的载体。

Summary

CISA added two actively exploited vulnerabilities to its KEV catalog: CVE-2025-34291 (Langflow, CVSS 9.4, origin validation error leading to RCE) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7, directory traversal enabling code injection). Langflow flaw was exploited by Iranian state group MuddyWater for initial access.

摘要

CISA 将两个已遭主动利用的漏洞加入其 KEV 目录：CVE-2025-34291（Langflow，CVSS 9.4，来源验证错误导致远程代码执行）和 CVE-2026-34926（Trend Micro Apex One，CVSS 6.7，目录遍历导致代码注入）。Langflow 漏洞已被伊朗国家背景黑客组织 MuddyWater 用于初始访问。

Deep Note

Key Points

  • CISA added CVE-2025-34291 (Langflow, CVSS 9.4) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7) to KEV due to active exploitation.
  • Langflow vulnerability: origin validation error allowing arbitrary code execution; combines permissive CORS, lack of CSRF protection, and a code-execution-by-design endpoint.
  • Exploitation of Langflow exposes all sensitive access tokens and API keys stored in the workspace, enabling cascading compromise of downstream services.
  • MuddyWater (Iranian state-sponsored group) exploited CVE-2025-34291 for initial access (per Ctrl-Alt-Intel, March 2026).
  • Trend Micro Apex One vulnerability: directory traversal allowing pre-authenticated local attacker to modify a key table and inject malicious code to agents.
  • Trend Micro confirmed at least one instance of active exploitation in the wild.
  • FCEB agencies must patch by June 4, 2026.

Context & Contrast

This incident marks a shift from theoretical AI supply-chain risks to real-world exploitation by a state-sponsored actor (MuddyWater) targeting an AI orchestration tool (Langflow) for initial access. Unlike previous AI vulnerabilities focused on model poisoning or prompt injection, this is a classic web/API security flaw (CORS + CSRF + code-exec endpoint) in an AI platform, bridging the gap between AI security and traditional web application security. The Trend Micro Apex One vulnerability is a more conventional on-premise endpoint security flaw, but its addition to KEV underscores that security products themselves remain a viable attack vector.

Why It Matters

For AI Security: This is a concrete example of an AI infrastructure component (Langflow) being used as an initial access vector by a state-sponsored APT, validating that AI tooling security is now a critical part of enterprise attack surface management. For SOC: The cascading compromise pattern (Langflow -> exposed tokens -> downstream cloud/SaaS) requires new detection logic for cross-environment lateral movement. For System Security: The Apex One vulnerability reinforces that endpoint protection platforms (EPP) themselves must be hardened against post-exploitation abuse, as they often have elevated privileges and broad system access.

Critical Take

The Langflow CVE-2025-34291 CVSS score of 9.4 is justified, but the actual exploit complexity is lowered by the combination of three separate weaknesses (CORS, CSRF, code-exec endpoint) that together create a chain. The attribution to MuddyWater is based on a single analysis (Ctrl-Alt-Intel) and may be contested. For Apex One, the requirement of pre-existing admin credentials significantly limits the blast radius; it is not a remote code execution from an unauthenticated attacker. The KEV inclusion is a strong signal but does not quantify the breadth of exploitation.

What to Watch

  • Monitor for additional CVEs in AI orchestration and agent frameworks (Langflow, LangChain, AutoGPT, etc.) related to CORS, CSRF, and API key exposure.
  • Track MuddyWater TTPs for post-exploitation activity following Langflow compromise.
  • Watch for proof-of-concept exploits or broader exploitation campaigns targeting on-premise endpoint security products (Trend Micro, but also others).
  • Observe if CISA issues additional guidance for securing AI development and deployment platforms.
  • Monitor for similar cascading token-exfiltration attacks in cloud/SaaS environments originating from AI tooling.
深度笔记

要点

  • CISA 因存在主动利用，将 CVE-2025-34291（Langflow，CVSS 9.4）和 CVE-2026-34926（Trend Micro Apex One，CVSS 6.7）加入 KEV 目录。
  • Langflow 漏洞：来源验证错误导致任意代码执行；结合了宽松的 CORS、缺乏 CSRF 保护以及一个设计上可执行代码的端点。
  • 利用 Langflow 漏洞会暴露工作区中存储的所有敏感访问令牌和 API 密钥，导致下游服务级联沦陷。
  • MuddyWater（伊朗国家背景黑客组织）利用 CVE-2025-34291 进行初始访问（据 Ctrl-Alt-Intel 2026 年 3 月报告）。
  • Trend Micro Apex One 漏洞：目录遍历，允许预认证的本地攻击者修改关键表并向代理注入恶意代码。
  • Trend Micro 确认至少发现一例野外主动利用实例。
  • FCEB 机构需在 2026 年 6 月 4 日前完成修复。

背景与对比

此事件标志着从理论上的 AI 供应链风险向现实世界利用的转变，由国家背景行为者（MuddyWater）针对 AI 编排工具（Langflow）进行初始访问。与以往侧重于模型投毒或提示注入的 AI 漏洞不同，这是一个 AI 平台中的经典 Web/API 安全缺陷（CORS + CSRF + 代码执行端点），弥合了 AI 安全与传统 Web 应用安全之间的鸿沟。Trend Micro Apex One 漏洞是一个更传统的本地端点安全缺陷，但其被加入 KEV 凸显了安全产品本身仍然是可行的攻击向量。

重要性

对于 AI 安全：这是一个具体的例子，表明 AI 基础设施组件（Langflow）被国家背景 APT 用作初始访问向量，验证了 AI 工具安全现已成为企业攻击面管理的关键部分。对于 SOC：级联沦陷模式（Langflow -> 暴露的令牌 -> 下游云/SaaS）需要新的跨环境横向移动检测逻辑。对于系统安全：Apex One 漏洞强化了端点保护平台（EPP）本身必须针对后利用滥用进行加固，因为它们通常拥有高权限和广泛的系统访问能力。

批判性视角

Langflow 的 CVE-2025-34291 的 CVSS 评分 9.4 是合理的，但实际利用复杂度因三个独立弱点（CORS、CSRF、代码执行端点）的组合而降低，它们共同形成了一个利用链。对 MuddyWater 的归因基于单一分析（Ctrl-Alt-Intel），可能存在争议。对于 Apex One，需要预先存在的管理员凭据显著限制了影响范围；它并非来自未认证攻击者的远程代码执行。加入 KEV 是一个强烈信号，但并未量化利用的广泛程度。

后续关注

  • 关注 AI 编排和智能体框架（Langflow、LangChain、AutoGPT 等）中与 CORS、CSRF 和 API 密钥泄露相关的其他 CVE。
  • 追踪 MuddyWater 在利用 Langflow 后的后利用活动 TTP。
  • 关注针对本地端点安全产品（Trend Micro 及其他厂商）的 PoC 或更广泛的利用活动。
  • 观察 CISA 是否会发布针对 AI 开发和部署平台安全的额外指南。
  • 监控源自 AI 工具的云/SaaS 环境中类似的级联令牌窃取攻击。
