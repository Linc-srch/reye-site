CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

supply-chain-attack
启示 · Insight

For SOC researchers, this confirms that AI/LLM orchestration tools (Langflow) are now a primary initial access vector for APT groups, directly impacting the threat model for edge-cloud AI deployments. The cascading compromise of downstream SaaS/cloud tokens via a single Langflow exploit is a critical data privacy and supply-chain risk that must be benchmarked against HarmonyOS's AI agent isolation mechanisms. For System Security researchers, the Langflow vulnerability is a textbook case of insecure-by-design (permissive CORS, no CSRF, intentional code-exec endpoint) in a modern AI framework, highlighting the need for memory-safe and formally verified isolation primitives in AI runtimes. The Trend Micro Apex One flaw (CVE-2026-34926) is a classic privilege-escalation path via directory traversal in an on-premise endpoint security product, underscoring that even security software itself is a vulnerable attack surface.

对于 SOC 研究员，这证实了 AI/LLM 编排工具（Langflow）现已成为 APT 组织的主要初始访问向量，直接影响端云 AI 部署的威胁模型。通过单个 Langflow 漏洞利用导致下游 SaaS/云令牌的级联泄露，是一个关键的数据隐私和供应链风险，必须与鸿蒙系统的 AI 智能体隔离机制进行对标。对于系统安全研究员，Langflow 漏洞是现代 AI 框架中典型的不安全设计案例（宽松的 CORS、无 CSRF、设计上允许代码执行的端点），凸显了在 AI 运行时中需要内存安全和形式化验证的隔离原语。Trend Micro Apex One 漏洞（CVE-2026-34926）是通过本地部署端点安全产品中的目录遍历实现的经典权限提升路径，强调了即使是安全软件本身也是一个易受攻击的攻击面。

Summary

CISA added two actively exploited vulnerabilities to its KEV catalog: CVE-2025-34291 (Langflow, CVSS 9.4, origin validation error leading to RCE) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7, directory traversal allowing code injection). Langflow flaw was exploited by Iranian state group MuddyWater for initial access.

摘要

CISA 将两个已遭积极利用的漏洞加入其 KEV 目录：CVE-2025-34291（Langflow，CVSS 9.4，来源验证错误导致远程代码执行）和 CVE-2026-34926（Trend Micro Apex One，CVSS 6.7，目录遍历允许代码注入）。Langflow 漏洞已被伊朗国家背景黑客组织 MuddyWater 用于初始访问。

Deep Note

Key Points

  • CISA added CVE-2025-34291 (Langflow, CVSS 9.4) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7) to KEV.
  • Langflow vulnerability: origin validation error, permissive CORS, lack of CSRF protection, and a code-execution endpoint by design.
  • Exploitation of Langflow exposes all sensitive access tokens and API keys, enabling cascading compromise of downstream services.
  • MuddyWater (Iranian state-sponsored group) exploited CVE-2025-34291 for initial access.
  • Trend Micro Apex One vulnerability: directory traversal allowing pre-authenticated local attacker to modify server key table and inject malicious code to agents.
  • Trend Micro confirmed at least one instance of active exploitation in the wild.
  • FCEB agencies must patch by June 4, 2026.

Context & Contrast

This is a significant escalation: while AI/ML framework vulnerabilities have been disclosed before, this is one of the first clear cases of a state-sponsored APT group weaponizing an AI orchestration tool (Langflow) for initial access, moving AI supply-chain attacks from theoretical to operational. The Trend Micro Apex One flaw is a reminder that on-premise security products remain a persistent attack surface, contrasting with the industry's shift to cloud-native endpoint detection and response (EDR).

Why It Matters

For AI Security, this demonstrates that AI orchestration layers are now a prime target for advanced persistent threats, necessitating robust security-by-design (not just bolted-on) for any AI agent framework. For OS/Endpoint Security, it shows that even security software (Apex One) can be a vector for privilege escalation, reinforcing the need for kernel-level isolation and integrity measurement. The cascading token theft from Langflow is a critical data privacy and supply-chain risk that SOC teams must model in their threat intelligence.

Critical Take

The Langflow CVE-2025-34291 is a design-level flaw (permissive CORS, no CSRF, intentional code-exec endpoint), not a subtle memory corruption bug, making it a 'low-hanging fruit' for attackers. The MuddyWater attribution is based on a single analysis (Ctrl-Alt-Intel) and should be treated with standard attribution confidence caveats. The Trend Micro Apex One flaw requires prior administrative credentials, significantly reducing its practical exploitability in well-secured environments.

What to Watch

  • Monitor for additional APT groups adopting AI orchestration tools (Langflow, Ollama, Flowise) as initial access vectors.
  • Track CISA KEV additions for other AI/ML framework vulnerabilities (e.g., Ollama OOB read mentioned in sidebar).
  • Observe how Langflow and similar projects harden their default configurations (CORS, CSRF, authentication).
  • Watch for patch bypasses or incomplete fixes for CVE-2026-34926 in on-premise Apex One deployments.
  • Follow Obsidian Security's research for deeper analysis of AI supply-chain attack chains.

背景与对比

这是一个显著的升级：虽然 AI/ML 框架漏洞此前已有披露，但这是首批明确的国家背景 APT 组织将 AI 编排工具（Langflow）武器化用于初始访问的案例之一，将 AI 供应链攻击从理论推向了实战。Trend Micro Apex One 漏洞则提醒我们，本地部署的安全产品仍然是一个持续存在的攻击面，这与行业向云原生端点检测与响应（EDR）的转变形成对比。

重要性

对于 AI 安全，这表明 AI 编排层现已成为高级持续性威胁的主要目标，任何 AI 智能体框架都需要强健的、内建安全（而非事后附加）的设计。对于操作系统/端点安全，它表明即使是安全软件（Apex One）也可能成为权限提升的向量，强化了内核级隔离和完整性度量的必要性。Langflow 的级联令牌窃取是一个关键的数据隐私和供应链风险，SOC 团队必须在其威胁情报中对此进行建模。

批判性视角

Langflow 的 CVE-2025-34291 是一个设计层面的缺陷（宽松的 CORS、无 CSRF、设计上允许代码执行的端点），而非微妙的 memory corruption 漏洞，使其成为攻击者的‘低垂果实’。MuddyWater 的归因基于单一分析（Ctrl-Alt-Intel），应遵循标准的归因置信度警告。Trend Micro Apex One 漏洞需要预先拥有管理凭据，这大大降低了其在良好安全环境中的实际可利用性。

后续关注

  • 观察 Langflow 及类似项目如何强化其默认配置（CORS、CSRF、身份验证）。
  • 关注 CVE-2026-34926 在本地部署 Apex One 中是否存在补丁绕过或不完整修复。
  • 跟进 Obsidian Security 的研究，以获取对 AI 供应链攻击链的更深入分析。
