CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

CISA Adds Exploited Langflow and Trend Micro Apex One Vulnerabilities to KEV

supply-chain-attack
启示 · Insight

For SOC researchers, this confirms that AI/LLM orchestration frameworks (Langflow) are now a prime initial-access vector for state-sponsored groups, directly impacting the AI security landscape. The cascading compromise of downstream cloud/SaaS tokens via a single Langflow instance is a critical threat model for edge-cloud environments. For System Security researchers, the Trend Micro Apex One vulnerability (CVE-2026-34926) highlights a classic on-premise supply-chain attack: a local admin compromise on the management server leads to agent-wide code injection, underscoring the need for strict server-side access controls and integrity checks in endpoint management platforms.

对于 SOC 研究员，这证实了 AI/LLM 编排框架（Langflow）现已成为国家背景黑客组织的首要初始访问向量，直接影响 AI 安全态势。单个 Langflow 实例被攻陷导致下游云/SaaS 令牌级联泄露，是端云环境中的关键威胁模型。对于系统安全研究员，Trend Micro Apex One 漏洞（CVE-2026-34926）凸显了经典本地供应链攻击：管理服务器上的本地管理员失陷导致代理端代码注入，强调了在终端管理平台中实施严格服务器端访问控制与完整性检查的必要性。

Summary

CISA added two actively exploited vulnerabilities to its KEV catalog: CVE-2025-34291 (Langflow, CVSS 9.4, origin validation error leading to RCE) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7, directory traversal allowing code injection). Langflow flaw was exploited by Iranian state group MuddyWater.

摘要

CISA 将两个已遭积极利用的漏洞加入其 KEV 目录：CVE-2025-34291（Langflow，CVSS 9.4，来源验证错误导致远程代码执行）和 CVE-2026-34926（Trend Micro Apex One，CVSS 6.7，目录遍历允许代码注入）。Langflow 漏洞已被伊朗国家背景黑客组织 MuddyWater 利用。

Deep Note

Key Points

  • CISA added CVE-2025-34291 (Langflow, CVSS 9.4) and CVE-2026-34926 (Trend Micro Apex One, CVSS 6.7) to KEV.
  • Langflow vulnerability: origin validation error, overly permissive CORS, lack of CSRF protection, and a code-execution-by-design endpoint.
  • Exploitation of Langflow exposes all stored API keys and tokens, enabling cascading compromise of downstream services.
  • MuddyWater (Iranian state-sponsored group) exploited CVE-2025-34291 for initial access (per Ctrl-Alt-Intel, March 2026).
  • Trend Micro Apex One vulnerability: directory traversal allowing malicious code injection into agents; requires prior admin credentials on the server.
  • FCEB agencies must patch by June 4, 2026.

Context & Contrast

This is a rare public attribution of an AI orchestration framework vulnerability to a state-sponsored APT group (MuddyWater), moving AI security from theoretical risk to confirmed operational threat. Prior Langflow vulnerabilities were mostly researched in academic or bug-bounty contexts. The Trend Micro Apex One flaw is a classic on-premise supply-chain attack vector, contrasting with the cloud-native Langflow attack, showing that both legacy and modern platforms are under active exploitation.

Why It Matters

For AI security, this demonstrates that AI agent orchestration layers are now a high-value, actively targeted attack surface, not just a theoretical concern. The cascading token theft model is a new class of supply-chain risk for AI deployments. For endpoint security, the Apex One case reinforces that on-premise management servers remain a critical weak link; a single admin compromise can pivot to mass agent compromise, bypassing endpoint defenses.

Critical Take

The Langflow CVE-2025-34291 exploit chain (CORS + no CSRF + code execution endpoint) is a basic web security failure, not an advanced AI-specific vulnerability. The hype around 'AI agent security' should be tempered: the root cause is poor web application security hygiene. The MuddyWater attribution relies on a single third-party analysis (Ctrl-Alt-Intel) and may not be independently confirmed. For Trend Micro, the pre-authentication requirement (admin credentials) significantly reduces the real-world exploitability in well-managed environments.

What to Watch

  • Monitor for additional Langflow CVEs or similar flaws in other AI orchestration tools (e.g., Flowise, Dify, AutoGPT).
  • Track MuddyWater's TTPs post-compromise to see if they deploy custom backdoors or use the access for data exfiltration.
  • Watch for CISA KEV additions involving AI/ML frameworks as a leading indicator of APT interest.
  • Observe Trend Micro's patch adoption rate for on-premise Apex One; slow patching could lead to broader exploitation.
深度笔记

要点

  • CISA 将 CVE-2025-34291（Langflow，CVSS 9.4）和 CVE-2026-34926（Trend Micro Apex One，CVSS 6.7）加入 KEV。
  • Langflow 漏洞：来源验证错误、过度宽松的 CORS、缺少 CSRF 保护以及一个设计上可执行代码的端点。
  • 利用 Langflow 会暴露所有存储的 API 密钥和令牌，导致下游服务级联失陷。
  • MuddyWater（伊朗国家背景黑客组织）利用 CVE-2025-34291 获取初始访问权限（据 Ctrl-Alt-Intel 2026 年 3 月报告）。
  • Trend Micro Apex One 漏洞：目录遍历，允许向代理注入恶意代码；需要服务器上的管理员凭据。
  • FCEB 机构需在 2026 年 6 月 4 日前完成修补。

背景与对比

这是罕见的将 AI 编排框架漏洞归因于国家背景 APT 组织（MuddyWater）的公开案例，将 AI 安全从理论风险推向了已确认的作战威胁。此前 Langflow 漏洞多在学术或漏洞奖励计划中被研究。Trend Micro Apex One 漏洞是经典的本地供应链攻击向量，与云原生的 Langflow 攻击形成对比，表明传统与现代平台均面临积极利用。

重要性

对于 AI 安全，这表明 AI 智能体编排层现在是一个高价值、被积极攻击的攻击面，而不仅仅是理论上的担忧。级联令牌窃取模型是 AI 部署中一类新的供应链风险。对于终端安全，Apex One 案例强化了本地管理服务器仍然是关键薄弱环节；一次管理员失陷可导致大规模代理失陷，绕过终端防御。

批判性视角

Langflow CVE-2025-34291 的利用链（CORS + 无 CSRF + 代码执行端点）是基本的 Web 安全失败，而非高级的 AI 特有漏洞。对 'AI 智能体安全' 的炒作应有所降温：根本原因是糟糕的 Web 应用安全实践。MuddyWater 的归因仅依赖单一第三方分析（Ctrl-Alt-Intel），可能未经独立确认。对于 Trend Micro，预认证要求（管理员凭据）显著降低了在管理良好环境中的实际可利用性。

后续关注

  • 关注 Langflow 的其他 CVE 或其他 AI 编排工具（如 Flowise、Dify、AutoGPT）中的类似漏洞。
  • 追踪 MuddyWater 在入侵后的 TTPs，观察其是否部署自定义后门或利用访问权限进行数据窃取。
  • 关注 CISA KEV 中涉及 AI/ML 框架的添加，作为 APT 兴趣的领先指标。
  • 观察 Trend Micro 对本地 Apex One 的补丁采用率；补丁缓慢可能导致更广泛的利用。
