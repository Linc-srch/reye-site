When Identity is the Attack Path

When Identity is the Attack Path

content-management digital-publishing supply-chain-attack
Summary

The article argues that identity itself has become the primary attack path in modern hybrid environments. It illustrates how a single cached AWS access key on a Windows machine could lead to 98% of cloud entities, and how forgotten role assignments and excessive permissions create exploitable chains from low-level footholds to critical assets. Despite rising identity-based breaches—responsible for nearly 90% of incidents in 2025—traditional tools like IGA and PAM operate in isolation, failing to map how exposures link across endpoints, Active Directory, and cloud. The author calls for unified visibility into identity, permissions, and environment context to close attack paths before attackers exploit them.

摘要

本文指出身份本身已成为现代混合环境中的主要 attack path。以缓存的 AWS 访问密钥为例，单个密钥可能通向 98% 的云实体。被遗忘的角色分配和过度权限可形成从低级入口到关键资产的链式暴露。尽管 2025 年近 90% 的事件与身份弱点相关，但传统的 IGA 和 PAM 工具孤立运作，无法跨端点、Active Directory 和云环境映射暴露链条。作者呼吁将身份、权限和环境上下文统一可视化，以在攻击者利用前关闭攻击路径。
