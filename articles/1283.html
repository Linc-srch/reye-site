Ghost CMS CVE-2026-26980 Exploited to Hijack 700+ Sites for ClickFix Attacks

Ghost CMS CVE-2026-26980 Exploited to Hijack 700+ Sites for ClickFix Attacks

content-management supply-chain-attack zero-trust
启示 · Insight

For a system security researcher, this incident is a textbook case of a web-facing vulnerability (SQLi) enabling a full server-side compromise (Admin API key theft), which then facilitates a client-side social engineering attack (ClickFix) to achieve code execution on end-user machines. The attack chain bridges web application security, content management system (CMS) hardening, and endpoint malware delivery. The use of a commercial cloaking service (Adspect) to evade analysis and target only real victims is a notable operational security (OPSEC) technique that complicates detection and reverse engineering. The final payload, a modified Electron app (Grape client) for persistence and C2, highlights the trend of using legitimate frameworks for malware.

对于系统安全研究员而言，此事件是一个教科书式的案例：一个面向 Web 的漏洞（SQL 注入）实现了完整的服务器端攻陷（管理员 API 密钥窃取），进而促成客户端社会工程攻击（ClickFix），最终在终端用户机器上实现代码执行。攻击链连接了 Web 应用安全、内容管理系统（CMS）加固和端点恶意软件投递。使用商业流量分发服务（Adspect）来逃避分析并仅针对真实受害者，是一项值得注意的运营安全（OPSEC）技术，使检测和逆向工程复杂化。最终载荷是一个用于持久化和 C2 通信的修改版 Electron 应用（Grape 客户端），凸显了使用合法框架进行恶意软件投递的趋势。

Summary

Threat actors are exploiting CVE-2026-26980, a critical SQL injection vulnerability in Ghost CMS (patched in Feb 2026), to steal Admin API keys and inject malicious JavaScript into over 700 websites, primarily to serve ClickFix fake CAPTCHA attacks that ultimately deploy remote access trojans.

摘要

威胁行为者正在利用 Ghost CMS 中一个关键的 SQL 注入漏洞 CVE-2026-26980（已于 2026 年 2 月修复），窃取管理员 API 密钥，并向超过 700 个网站注入恶意 JavaScript，主要用于发起 ClickFix 虚假验证码攻击，最终部署远程访问木马。

Deep Note

Key Points

  • CVE-2026-26980 is an unauthenticated SQL injection in Ghost CMS's Content API (CVSS 9.4), discovered by Anthropic using Claude.
  • Patched in Ghost CMS version 6.19.1 (February 2026).
  • Exploitation allows theft of the site's Admin API key, enabling bulk modification of articles.
  • Attackers injected a two-stage JavaScript loader at the bottom of articles.
  • The loader retrieves a PHP script from 'clo4shara[.]xyz' which uses Adspect cloaking to fingerprint visitors and serve fake CAPTCHA (ClickFix) only to real users.
  • ClickFix attack instructs victims to run a Base64-encoded command, leading to a DLL or JavaScript payload that drops a modified Electron app (Grape client) for persistence and C2.
  • Over 700 sites compromised, targeting universities, blockchain, AI, SaaS, security research, media, and fintech.
  • First detected May 7, 2026; at least two threat clusters involved.

Context & Contrast

This attack is a sophisticated evolution of the classic 'watering hole' technique, leveraging a CMS vulnerability for mass compromise rather than targeted site defacement. The use of a commercial cloaking service (Adspect) to filter out researchers and sandboxes is a significant step up in operational security compared to typical drive-by download campaigns. The final payload being a modified Electron app (Grape) for persistence is a departure from traditional malware droppers, showing an adaptation to modern endpoint defenses by using a legitimate framework. The scale (700+ sites) and speed (compromise within a day) are notable.

Why It Matters

This incident demonstrates a critical supply-chain-like risk: a vulnerability in a widely-used CMS (Ghost) can be weaponized to compromise hundreds of downstream sites, which then serve as trusted vectors to attack their visitors. For system security, it highlights the importance of patching web-facing software (especially CMS platforms) and the need for robust API security (authentication, authorization, input validation). The attack chain also underscores the convergence of web security and endpoint security, where a web vulnerability leads directly to a sophisticated client-side malware deployment. The use of AI (Claude) to discover the vulnerability is a notable trend.

Critical Take

The report is from a single vendor (QiAnXin XLab) and while detailed, independent verification of the scale (700+ sites) and attribution (two clusters) is pending. The vulnerability was patched three months prior to the observed exploitation, suggesting a significant window of exposure for unpatched instances. The effectiveness of the ClickFix attack relies on user social engineering, which is a variable factor. The claim that the vulnerability was discovered by AI (Claude) is interesting but does not change the technical analysis of the exploit. The long-term impact on compromised users is unverified.

What to Watch

  • Monitor for Ghost CMS CVE-2026-26980 exploitation attempts in honeypots and web logs.
  • Track the evolution of the ClickFix payload, particularly the modified Grape Electron app and its C2 infrastructure.
  • Watch for similar attacks using other CMS platforms (WordPress, Drupal) with SQL injection or API key theft vulnerabilities.
  • Observe the adoption of commercial cloaking services (like Adspect) in other malware campaigns.
  • Look for disclosure of the technical details of the SQL injection vulnerability (CVE-2026-26980) for deeper analysis.
深度笔记

要点

  • CVE-2026-26980 是 Ghost CMS 内容 API 中的一个未经认证的 SQL 注入漏洞（CVSS 9.4），由 Anthropic 使用 Claude 发现。
  • 已在 Ghost CMS 6.19.1 版本中修复（2026 年 2 月）。
  • 利用该漏洞可窃取网站的管理员 API 密钥，从而批量修改文章。
  • 攻击者在文章底部注入了两阶段 JavaScript 加载器。
  • 加载器从 'clo4shara[.]xyz' 获取 PHP 脚本，该脚本使用 Adspect 流量分发服务对访问者进行指纹识别，并仅向真实用户提供虚假验证码（ClickFix）。
  • ClickFix 攻击指示受害者运行 Base64 编码的命令，导致 DLL 或 JavaScript 载荷，最终投放一个修改版的 Electron 应用（Grape 客户端）以实现持久化和 C2 通信。
  • 超过 700 个网站被攻陷，目标涵盖大学、区块链、人工智能、SaaS、安全研究、媒体和金融科技领域。
  • 首次检测于 2026 年 5 月 7 日；至少涉及两个威胁集群。

背景与对比

此攻击是经典‘水坑’技术的一次复杂演变，利用 CMS 漏洞进行大规模攻陷，而非针对性的网站篡改。与典型的路过式下载攻击相比，使用商业流量分发服务（Adspect）来过滤研究人员和沙箱，在运营安全方面是一个显著的进步。最终载荷是一个用于持久化的修改版 Electron 应用（Grape），这背离了传统的恶意软件投放器，显示出通过使用合法框架来适应现代端点防御的趋势。其规模（超过 700 个网站）和速度（一天内完成攻陷）值得注意。

重要性

此事件展示了一种关键的类似供应链的风险：广泛使用的 CMS（Ghost）中的漏洞可被武器化，以攻陷数百个下游网站，这些网站随后成为攻击其访客的可信向量。对于系统安全而言，它凸显了修补面向 Web 的软件（尤其是 CMS 平台）的重要性，以及健壮的 API 安全（认证、授权、输入验证）的必要性。该攻击链还强调了 Web 安全与端点安全的融合，即一个 Web 漏洞直接导致了复杂的客户端恶意软件部署。使用 AI（Claude）发现漏洞是一个值得注意的趋势。

批判性视角

该报告来自单一厂商（奇安信 XLab），虽然详细，但关于规模（超过 700 个网站）和归因（两个集群）的独立验证尚待进行。该漏洞在观察到被利用前三个月已修复，表明未修补实例存在显著的暴露窗口。ClickFix 攻击的有效性依赖于用户的社会工程学，这是一个可变因素。关于漏洞由 AI（Claude）发现的声称很有趣，但并不改变对漏洞利用的技术分析。对受感染用户的长期影响尚未得到验证。

  • 关注针对其他 CMS 平台（WordPress、Drupal）利用 SQL 注入或 API 密钥窃取漏洞的类似攻击。
  • 观察商业流量分发服务（如 Adspect）在其他恶意软件活动中的使用情况。
  • 期待 CVE-2026-26980 SQL 注入漏洞的技术细节披露，以便进行更深入的分析。
