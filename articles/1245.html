Megalodon GitHub Attack Targets 5,561 Repos with Malicious CI/CD Workflows

Megalodon GitHub Attack Targets 5,561 Repos with Malicious CI/CD Workflows

supply-chain-attack Malware & Threats Supply Chain Security
Summary

A new automated campaign dubbed 'Megalodon' pushed 5,718 malicious commits to 5,561 GitHub repositories within six hours using throwaway accounts and forged author identities. The attacker injected GitHub Actions workflows containing base64-encoded bash payloads that exfiltrate CI secrets, cloud credentials, SSH keys, OIDC tokens, and source code secrets to a command-and-control server. The attack is linked to the TeamPCP group, which has been compromising open-source projects in a worm-like fashion. This incident highlights the escalating threat of supply chain attacks targeting CI/CD pipelines and credential theft at scale.

摘要

一项名为'Megalodon'的自动化攻击活动在六小时内向5,561个GitHub仓库推送了5,718个恶意提交，使用一次性账户和伪造的作者身份（如build-bot、auto-ci）注入GitHub Actions工作流，其中包含base64编码的bash载荷，用于窃取CI secret、云凭证、SSH密钥、OIDC token等敏感数据，并发送至C2服务器。该攻击与TeamPCP组织相关，其通过供应链攻击大规模传播恶意软件，凸显了CI/CD管道安全面临的严峻挑战。
