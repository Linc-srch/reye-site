npm Adds 2FA-Gated Publishing and Package Install Controls Against Supply Chain Attacks

npm Adds 2FA-Gated Publishing and Package Install Controls Against Supply Chain Attacks

content-management digital-publishing supply-chain-attack
Summary

GitHub has introduced staged publishing for npm, requiring maintainers to pass a two-factor authentication (2FA) challenge before a package is publicly released. This feature ensures human approval for every publish, even from automated CI/CD pipelines. Additionally, npm now supports install source flags—--allow-file, --allow-remote, and --allow-directory—giving developers explicit control over non-registry package installations. These measures counter the surge in software supply chain attacks targeting open-source ecosystems, such as those by the TeamPCP group. GitHub recommends pairing staged publishing with trusted publishing via OIDC for maximum security.

摘要

GitHub为npm推出了分阶段发布功能，要求维护者通过two-factor authentication (2FA)验证后才能公开发布包。此功能确保每次发布都有“人为在场”批准，包括来自CI/CD工作流的发布。同时，npm新增了--allow-file、--allow-remote和--allow-directory等安装源标志，让开发者对非注册表安装源进行显式控制。这些措施旨在应对针对开源生态系统的软件供应链攻击激增。
