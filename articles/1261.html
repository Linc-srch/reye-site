Packagist Supply Chain Attack Infects 8 Packages Using GitHub-Hosted Linux Malware

Packagist Supply Chain Attack Infects 8 Packages Using GitHub-Hosted Linux Malware

supply-chain-attack
Summary

A coordinated supply chain attack compromised eight packages on Packagist by injecting malicious code into package.json instead of composer.json, targeting projects that use JavaScript build tooling alongside PHP. The code downloads a Linux binary from a GitHub Releases URL, saves it to /tmp/.sshd, and executes it in the background. The same payload was found in 777 GitHub files, including workflow files, suggesting a broader campaign. The second-stage payload's purpose is unknown as the associated GitHub account is no longer available. This cross-ecosystem placement allows the attack to bypass traditional PHP dependency scans, highlighting the need for comprehensive security checks across all build artifacts.

摘要

Packagist供应链攻击入侵了8个包，通过将恶意代码注入package.json而非composer.json，利用JavaScript构建工具链下载并执行GitHub上的Linux二进制文件。该跨生态系统攻击绕过了PHP依赖扫描，相同载荷出现在777个GitHub文件中，表明这是一次大规模活动。由于托管仓库的GitHub账户已不可用，第二阶段载荷的性质不明。此事件凸显了软件供应链安全中跨平台检查的重要性。
