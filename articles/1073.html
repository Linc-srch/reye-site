1) Why-read

LMAC uses an LLM to iteratively design and refine a communication protocol for cooperative MARL, explicitly targeting state reconstruction accuracy and uniformity across agents.

2) CRGP

C — Context

Cooperative multi-agent reinforcement learning (MARL) often suffers from partial observability. Communication protocols have been introduced to mitigate this, but prior methods like TarMAC, SMS, and T2MAC do not explicitly optimize for how accurately and uniformly agents can reconstruct the global state from exchanged messages.

R — Related work

  • Broadcast communication methods (Sukhbaatar et al., 2016; Guan et al., 2022) share the same message with all agents.
  • Agent-wise communication methods (Das et al., 2019; Wang et al., 2019; Yuan et al., 2022) enable selective message exchange using attention (TarMAC), Shapley values (SMS), or evidence fusion (T2MAC).
  • LLMs have been used for reasoning (CoT, Reflexion) and for RL tasks such as reward design, state representation, and policy networks.
  • Prior LLM-based MARL communication (Li et al., 2024) requires online LLM interaction at each timestep, increasing cost and limiting applicability to text-based environments.

G — Gap

Existing communication methods in MARL do not explicitly address how efficiently and accurately agents can reconstruct the underlying global state from messages. Identifying which essential messages are needed for state recovery is challenging because it requires understanding the task objective and the relationship between state and observations, which prior learned protocols lack.

P — Proposal

LMAC leverages an LLM's reasoning capability to design a communication protocol by describing the task goal and state/observation dimensions in natural language. It uses Reflexion to generate an initial protocol and iteratively refines it via two criteria: recovery enhancement (improving state reconstruction accuracy) and imbalance mitigation (reducing information gaps across agents), using feedback derived solely from RL transition data without online LLM interaction.

4) Experiments

Main results

On StarCraft II after 2M timesteps, LMAC achieves lower average reconstruction error of enemy position and lower inter-agent variance of reconstruction error compared to baselines (TarMAC, SMS, T2MAC). Performance gains are reported across diverse MARL benchmarks, with substantial improvements over prior communication baselines.

Ablation

The two-step refinement (recovery enhancement and imbalance mitigation) is critical: removing either step degrades state reconstruction accuracy and increases inter-agent variance, confirming that both criteria contribute to uniform and accurate state recovery.

Limitations

["The approach relies on the LLM's understanding of the task and state dimensions, which may be brittle for highly complex or novel environments.", 'The iterative refinement uses RL transition data, but the protocol is fixed after training and may not adapt to dynamic changes during execution.', 'LLM usage, though offline, still incurs computational cost for protocol design and refinement.']

5) Why it matters

  1. Directly targets state reconstruction quality, a fundamental bottleneck in partially observable MARL, which aligns with research on representation learning and information bottlenecks.
  2. Demonstrates a practical use of LLMs for structured protocol design without online interaction, reducing cost and broadening applicability to general MARL environments.
  3. The explicit two-criterion refinement (accuracy + uniformity) provides a principled framework that could be extended to other multi-agent coordination problems.

6) Next steps

  • [ ] Test LMAC on more complex environments with higher-dimensional state spaces to assess scalability of LLM-based protocol design.
  • [ ] Explore integrating the learned protocol with dynamic adaptation mechanisms (e.g., meta-learning) to handle non-stationary tasks.
  • [ ] Investigate whether the protocol can be distilled into a lightweight neural network to eliminate LLM dependency at execution time.
