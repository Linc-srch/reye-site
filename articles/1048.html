Building a safe, effective sandbox to enable Codex on Windows

Building a safe, effective sandbox to enable Codex on Windows

2026-05-18 · openai_rss

Summary

OpenAI engineers built a custom sandbox for Codex on Windows after finding existing tools like AppContainer, Windows Sandbox, and Mandatory Integrity Control unsuitable. The solution uses SIDs and write-restricted tokens to constrain file writes and network access without requiring administrator privileges. This enables Codex to safely run developer workflows—shells, Git, Python, build tools—while preventing unauthorized modifications outside the workspace. The sandbox enforces read-only access outside the workspace and blocks internet access unless explicitly allowed, balancing safety and productivity for Windows users.

摘要

OpenAI工程师为Windows上的Codex构建了自定义沙箱，因AppContainer、Windows Sandbox和Mandatory Integrity Control等现有工具不适用。该方案利用SIDs和write-restricted tokens来限制文件写入和网络访问，无需管理员权限。沙箱允许Codex安全运行shell、Git、Python等开发工具，同时防止对工作区外的未授权修改，并默认阻止互联网访问，在安全性与生产力间取得平衡。
