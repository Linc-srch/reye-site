Laravel-Lang PHP Packages Compromised to Deliver Cross-Platform Credential Stealer

Laravel-Lang PHP Packages Compromised to Deliver Cross-Platform Credential Stealer

supply-chain-attack
Summary

Cybersecurity researchers have identified a supply chain attack targeting multiple Laravel-Lang PHP packages. The compromised packages include laravel-lang/lang, http-statuses, attributes, and actions. Over 700 malicious versions were published via automated mass tagging, likely due to compromised organization-level credentials. The attack embeds a backdoor in 'src/helpers.php' that fetches a cross-platform PHP credential stealer from an external server. The stealer collects cloud IAM roles, access tokens, crypto wallet keys, browser credentials, session tokens, and configuration files. It encrypts exfiltrated data with AES-256 and self-deletes. The campaign affects Windows, Linux, and macOS and highlights risks in open-source software supply chains.

摘要

网络安全研究人员发现了一起针对多个 Laravel-Lang PHP 包的软件供应链攻击。受影响的包包括 laravel-lang/lang、http-statuses、attributes 和 actions。攻击者通过自动化批量标记发布了超过700个恶意版本，疑似获取了组织级凭证。后门代码嵌入在 'src/helpers.php' 中，可获取跨平台的 PHP 凭据窃取器，窃取云 IAM 角色、访问令牌、加密货币钱包密钥、浏览器凭据、会话令牌及配置文件等。数据经 AES-256 加密后外传，并自删除以掩盖痕迹。此攻击影响 Windows、Linux 和 macOS，凸显了开源供应链的风险。
