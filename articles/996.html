1) Why-read

2) CRGP

C — Context

Deep research agents have made progress on complex information-seeking tasks, but single ReAct-style trajectories are limited to one sequential path. Recent state-of-the-art systems scale inference-time compute via parallel search and aggregation, but parallel rollouts often duplicate evidence rather than complete it, leading to diminishing returns and context window limits.

R — Related work

  • ReAct-style agents for web browsing and information seeking
  • Parallel trajectory sampling with majority voting, best-of-N selection, or LLM-based synthesis
  • Self-consistency and learned aggregation methods for multi-trajectory answers
  • RL-trained agent swarms for parallel search
  • Atomic-fact verification with multi-source corroboration

G — Gap

Existing parallel-agent methods treat deep research as a flat collection of linear traces, with no shared notion of which evidence pieces have been gathered, which support or contradict each other, and which are still missing. This leads to redundant evidence collection and saturating gains as parallel rollouts increase, while the aggregation context grows linearly with the number of trajectories.

P — Proposal

Argus introduces a Searcher that collects evidence traces via ReAct-style interaction and a Navigator that maintains a shared evidence graph, verifying missing pieces and dispatching targeted Searchers. The Navigator is trained with reinforcement learning to verify, dispatch, and synthesize, while the Searcher remains a standard ReAct agent, enabling scalable parallel execution without retraining.

4) Experiments

Main results

On a 35B-A3B MoE backbone, Argus improves over the raw Searcher by +5.5 points with a single Searcher and +12.7 points with 8 parallel Searchers averaged over eight benchmarks. With 64 Searchers, it reaches 86.2% on BrowseComp, surpassing all proprietary agents benchmarked, while the Navigator's reasoning context stays under 21.5K tokens.

Ablation

The evidence graph compression achieves a 1,200:1 ratio (25.6M tokens of Searcher output to 21.5K tokens for the Navigator), decoupling Navigator context from Searcher count and enabling scaling to 64 parallel Searchers.

Limitations

["The Navigator's graph construction and verification rely on the quality of the underlying LLM, which may introduce errors in node parsing or edge labeling.", 'The system assumes that evidence can be decomposed into independent pieces, which may not hold for highly interconnected or subjective questions.', 'The Searcher is stateless and does not see the graph, potentially missing opportunities for cross-referencing during its rollout.']

5) Why it matters

  1. Argus demonstrates a structured evidence assembly approach that avoids the diminishing returns of flat parallel rollouts, directly relevant to your work on scalable agentic systems.
  2. The 1,200:1 compression ratio shows how a shared graph representation can decouple reasoning context from the number of parallel workers, a key insight for building efficient multi-agent systems.
  3. The RL-trained Navigator with verify-dispatch-synthesize loop provides a principled way to orchestrate multiple agents without retraining the base Searcher, which could inspire your own agent coordination designs.
