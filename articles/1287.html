Summary

This paper argues that the next bottleneck in agentic AI is system scaling (the harness around the model) rather than model scaling alone, and proposes a framework and reference harness to study it.

摘要

本文提出智能体AI的下一个瓶颈是系统扩展（模型周围的系统架构），而非单纯的模型扩展。它识别了三个核心瓶颈——上下文治理、可信记忆和动态技能路由，并发布了Python原生参考架构CheetahClaws，以促进系统级设计选择的研究。

1) Why-read

This paper argues that the next bottleneck in agentic AI is system scaling (the harness around the model) rather than model scaling alone, and proposes a framework and reference harness to study it.

2) CRGP

C — Context

Recent progress in LLMs has enabled agents to use tools, retrieve information, and execute long-horizon workflows, but evaluation remains model-centric, focusing on final-task success while treating memory, retrieval, and orchestration as secondary. Production systems like Claude Code and OpenClaw show that practical capability emerges from the interaction between the model and the surrounding harness.

R — Related work

  • ReAct (Yao et al., 2022) and self-taught tool invocation (Schick et al., 2023) established tool-using language models.
  • SWE-agent (Yang et al., 2024) shows careful tool-schema design improves benchmark accuracy with a fixed backbone model.
  • MemGPT (Packer et al., 2023) and Voyager (Wang et al., 2023) study memory as a system component.
  • Multi-agent frameworks like AutoGen (Wu et al., 2024), MetaGPT (Hong et al., 2024), and CAMEL (Li et al., 2023) formalize agent-to-agent communication.
  • Kapoor et al. (2024) find that many agent benchmark results conflate capability with costs and prompting strategy.

G — Gap

Current agent evaluation is model-centric, reducing agents to final-task success while ignoring the harness components (memory, context construction, skill routing, orchestration, verification). This hides that reported model scores are actually model-plus-harness scores, and fails to measure process-level properties like trajectory quality, memory hygiene, and verification cost.

P — Proposal

The paper proposes a system-scaling framing where the agent harness (structured execution layer around the foundation model) is treated as a first-class object of design, evaluation, and optimization. It identifies three core bottlenecks—context governance, trustworthy memory, and dynamic skill routing—and outlines an evaluation agenda with metrics like trajectory quality, memory hygiene, context efficiency, and verification cost. A Python-native reference harness, CheetahClaws, is released for concrete comparison.

4) Experiments

Main results

The paper does not present quantitative experimental results; it is a position paper with a framework and reference harness. It references empirical findings: SWE-agent improved benchmark accuracy substantially with a fixed backbone model via tool-schema redesign; tau-bench shows agents strong under single-shot pass rates can collapse under pass^k, exposing a reliability gap.

Ablation

No ablation studies are reported in the paper.

Limitations

['The paper is a position/framework paper without empirical validation of the proposed harness-level metrics.', 'The reference harness CheetahClaws is introduced but not yet evaluated against existing systems on the proposed metrics.', 'The framework may require significant engineering effort to implement in practice.']

5) Why it matters

  1. Directly addresses the need to separate model capability from system design in agentic AI, which is crucial for reproducible research.
  2. Proposes concrete metrics (trajectory quality, memory hygiene, context efficiency) that align with evaluating long-horizon, safety-critical agent deployments.
  3. The harness-centric view can inform the design of auditable and verifiable agent architectures, relevant to system safety and AI safety domains.

  • [ ] Investigate context governance strategies (retrieval, compression, ordering) to mitigate attention dilution in long-horizon tasks.

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：LLM的近期进展使智能体能够使用工具、检索信息并执行长周期工作流，但评估仍以模型为中心，关注最终任务成功，而将记忆、检索和编排视为次要。生产系统如Claude Code和OpenClaw表明，实际能力源于模型与周围系统架构的交互。

相关工作：ReAct和自学习工具调用建立了工具使用语言模型；SWE-agent显示精心设计的工具模式可提高固定骨干模型的基准准确率；MemGPT和Voyager将记忆作为系统组件研究；多智能体框架如AutoGen、MetaGPT和CAMEL形式化了智能体间通信；Kapoor等人发现许多智能体基准结果混淆了能力与成本和提示策略。

差距：当前智能体评估以模型为中心，将智能体简化为最终任务成功，忽略了系统架构组件（记忆、上下文构建、技能路由、编排、验证）。这掩盖了报告的模型分数实际上是模型加系统架构的分数，并且未能衡量过程级属性如轨迹质量、记忆卫生和验证成本。

提议：本文提出系统扩展框架，将智能体系统架构（基础模型周围的结构化执行层）视为设计、评估和优化的一等对象。它识别了三个核心瓶颈——上下文治理、可信记忆和动态技能路由，并概述了包含轨迹质量、记忆卫生、上下文效率和验证成本等指标的评估议程。发布了Python原生参考架构CheetahClaws以进行具体比较。

3) 实验

本文是一篇立场论文，未呈现定量实验结果。它引用了实证发现：SWE-agent通过工具模式重新设计，在固定骨干模型下显著提高了基准准确率；tau-bench显示在单次通过率上表现强劲的智能体在pass^k下可能崩溃，暴露了可靠性差距。
