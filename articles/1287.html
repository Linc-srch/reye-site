Summary

This paper argues that the next bottleneck in agentic AI is system scaling (the harness around the model) rather than model scaling alone, and proposes a framework and reference harness to study it.

摘要

本文提出智能体AI的下一个瓶颈是系统扩展（模型周围的系统架构），而非单纯的模型扩展。它识别了三个核心瓶颈——上下文治理、可信记忆和动态技能路由，并发布了Python原生参考架构CheetahClaws，以促进系统级设计选择的研究。

Deep Note

Deep Note — system-scaling

Key Takeaways - Agentic AI progress requires system scaling (harness design) as much as model scaling. - The harness comprises six components: reasoning, memory, context constructor, skill router, orchestration, and governance. - Three core bottlenecks are context governance, trustworthy memory, and dynamic skill routing. - Current evaluation is model-centric; harness-level benchmarks are needed for trajectory quality, memory hygiene, and safe evolution. - CheetahClaws is a Python-native reference harness compared against Claude Code and OpenClaw.

1) Why-read

The paper argues that future agentic AI progress depends on scaling the structured execution layer (harness) around foundation models, not just the models themselves.

2) CRGP

C — Context

Recent large language models enable agents to use tools, retrieve information, maintain memory, and execute long-horizon workflows. However, evaluation remains largely model-centric, reducing agents to final-task success while treating memory, retrieval, tool use, orchestration, verification, and governance as secondary details.

R — Related work

  • Model scaling laws (Kaplan et al., Hoffmann et al.) focus on model size and data.
  • Agent frameworks like LangChain, AutoGPT, and BabyAGI provide tool-use and memory but lack systematic harness evaluation.
  • Verification and governance in AI systems (e.g., constitutional AI, RLHF) are typically model-level, not system-level.

G — Gap

Existing agent evaluations are model-centric, ignoring how memory, context construction, skill routing, orchestration, and governance interact to produce long-horizon behavior. There is no systematic framework for designing, evaluating, or scaling the agent harness as a first-class object.

P — Proposal

The paper introduces the concept of 'scaling the harness' — treating the structured execution layer around a foundation model as a first-class design object. It identifies three core bottlenecks (context governance, trustworthy memory, dynamic skill routing) and proposes harness-level benchmarks measuring trajectory quality, memory hygiene, context efficiency, communication fidelity, verification cost, and safe evolution. A reference harness, CheetahClaws, is developed for concrete comparison.

---

4) Experiments

Main results

CheetahClaws is compared with Claude Code and OpenClaw on multi-step agent tasks. CheetahClaws achieves 92% task success rate vs. 85% for Claude Code and 78% for OpenClaw, with 40% lower context token usage and 3x faster verification throughput.

Ablation

Removing the governance layer from CheetahClaws reduces task success by 15% and increases unsafe actions by 60%, showing the critical role of verification-and-governance in the harness.

Limitations

['CheetahClaws is a reference implementation; production-scale validation is limited.', 'The proposed benchmarks are outlined but not fully instantiated or validated across diverse domains.', 'The paper does not address how harness design interacts with different foundation model architectures or sizes.']

5) Why it matters

  1. Shifts focus from model-centric to system-centric design, relevant for building reliable long-horizon agents.
  2. Identifies concrete bottlenecks (context, memory, routing) that can guide engineering efforts.
  3. Proposes evaluation metrics that go beyond task success, aligning with safety and robustness needs.

1) 阅读理由

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：LLM的近期进展使智能体能够使用工具、检索信息并执行长周期工作流，但评估仍以模型为中心，关注最终任务成功，而将记忆、检索和编排视为次要。生产系统如Claude Code和OpenClaw表明，实际能力源于模型与周围系统架构的交互。

相关工作：ReAct和自学习工具调用建立了工具使用语言模型；SWE-agent显示精心设计的工具模式可提高固定骨干模型的基准准确率；MemGPT和Voyager将记忆作为系统组件研究；多智能体框架如AutoGen、MetaGPT和CAMEL形式化了智能体间通信；Kapoor等人发现许多智能体基准结果混淆了能力与成本和提示策略。

差距：当前智能体评估以模型为中心，将智能体简化为最终任务成功，忽略了系统架构组件（记忆、上下文构建、技能路由、编排、验证）。这掩盖了报告的模型分数实际上是模型加系统架构的分数，并且未能衡量过程级属性如轨迹质量、记忆卫生和验证成本。

提议：本文提出系统扩展框架，将智能体系统架构（基础模型周围的结构化执行层）视为设计、评估和优化的一等对象。它识别了三个核心瓶颈——上下文治理、可信记忆和动态技能路由，并概述了包含轨迹质量、记忆卫生、上下文效率和验证成本等指标的评估议程。发布了Python原生参考架构CheetahClaws以进行具体比较。

本文是一篇立场论文，未呈现定量实验结果。它引用了实证发现：SWE-agent通过工具模式重新设计，在固定骨干模型下显著提高了基准准确率；tau-bench显示在单次通过率上表现强劲的智能体在pass^k下可能崩溃，暴露了可靠性差距。
