Microsoft Warns of Two Actively Exploited Defender Vulnerabilities

Microsoft Warns of Two Actively Exploited Defender Vulnerabilities

supply-chain-attack
启示 · Insight

For a system security researcher, these vulnerabilities are significant because they target the kernel-adjacent attack surface of Microsoft Defender, a critical security component that runs with high privileges (SYSTEM). The privilege escalation via improper link resolution (CVE-2026-41091) is a classic 'link following' bug, a recurring pattern in Windows security that can be exploited for local privilege escalation (LPE). The heap-based buffer overflow (CVE-2026-45584) in Defender, though not yet exploited, represents a remote code execution (RCE) vector in a component that parses untrusted data, making it a high-value target for exploitation research. The active exploitation of these flaws underscores the importance of fuzzing and static analysis on security products themselves, as they are a prime target for attackers seeking to disable or subvert endpoint defenses.

对于系统安全研究人员来说，这些漏洞意义重大，因为它们针对的是Microsoft Defender这个内核相邻的攻击面，这是一个以高权限（SYSTEM）运行的关键安全组件。通过不当链接解析实现的权限提升（CVE-2026-41091）是一个经典的“链接跟随”漏洞，这是Windows安全中反复出现的模式，可用于本地权限提升（LPE）。Defender中的堆缓冲区溢出（CVE-2026-45584）虽然尚未被利用，但代表了一个在解析不可信数据的组件中的远程代码执行（RCE）向量，使其成为利用研究的高价值目标。这些漏洞的积极利用凸显了对安全产品本身进行模糊测试和静态分析的重要性，因为它们是攻击者试图禁用或破坏端点防御的主要目标。

Summary

Microsoft disclosed two actively exploited vulnerabilities in Microsoft Defender: a privilege escalation flaw (CVE-2026-41091, CVSS 7.8) and a denial-of-service bug (CVE-2026-45498, CVSS 4.0), both patched in platform updates 1.1.26040.8 and 4.18.26040.7. CISA added both to its KEV catalog, and researchers linked them to previously disclosed zero-days (RedSun and UnDefend).

摘要

微软披露了Microsoft Defender中两个被积极利用的漏洞：一个权限提升漏洞（CVE-2026-41091，CVSS 7.8）和一个拒绝服务漏洞（CVE-2026-45498，CVSS 4.0），已在平台更新1.1.26040.8和4.18.26040.7中修复。CISA已将两者列入其KEV目录，研究人员将其与之前披露的零日漏洞（RedSun和UnDefend）联系起来。

Deep Note

Key Points

  • Two actively exploited Microsoft Defender vulnerabilities: CVE-2026-41091 (privilege escalation, CVSS 7.8) and CVE-2026-45498 (denial-of-service, CVSS 4.0).
  • CVE-2026-41091 is a 'link following' bug allowing local privilege escalation to SYSTEM.
  • A third, un-exploited heap-based buffer overflow (CVE-2026-45584, CVSS 8.1) could allow remote code execution.
  • Patches are in Defender Antimalware Platform versions 1.1.26040.8 and 4.18.26040.7; updates are automatic.
  • CISA added both exploited flaws to its Known Exploited Vulnerabilities (KEV) catalog.
  • Researchers linked the exploited flaws to previously disclosed zero-days (RedSun and UnDefend) by Chaotic Eclipse.
  • Systems with Defender disabled are not vulnerable.

Context & Contrast

This is not the first time Microsoft Defender has been found vulnerable; it has a history of critical flaws (e.g., CVE-2021-1647, CVE-2022-24521). However, the active exploitation of a privilege escalation bug in a security product is particularly notable because it directly undermines the trust placed in endpoint protection. The linkage to publicly disclosed zero-days (RedSun, UnDefend) suggests a trend where vulnerability research groups are increasingly targeting security software, and that exploit code may be rapidly weaponized by threat actors. The inclusion of decades-old IE and DirectX flaws in the same KEV update highlights the persistent risk from legacy code, contrasting with the modern attack surface of Defender.

Why It Matters

These vulnerabilities matter because they demonstrate that security software itself is a critical attack surface. A successful exploit of Defender can disable or bypass the primary defense mechanism on millions of Windows endpoints, enabling further compromise. The privilege escalation bug (CVE-2026-41091) is a direct path to SYSTEM, the highest privilege level on Windows, which is a key goal for malware and post-exploitation toolkits. The heap overflow (CVE-2026-45584) is a potential wormable RCE vector in a component that processes network traffic or files, making it a high-priority target for patch analysis and exploit development research.

Critical Take

The article relies heavily on Microsoft's advisories and researcher claims. The exact exploitation details (e.g., attack vector, required user interaction) for CVE-2026-41091 and CVE-2026-45498 are not fully described. The linkage to 'RedSun' and 'UnDefend' is based on overlapping descriptions, not confirmed by Microsoft. The denial-of-service bug (CVE-2026-45498) has a low CVSS score (4.0), suggesting its impact may be limited, but its inclusion in the KEV catalog indicates real-world weaponization. The claim that no action is required for updates is true for default configurations, but enterprise-managed systems may have delayed update policies.

What to Watch

  • Public release of proof-of-concept (PoC) exploits for CVE-2026-41091 and CVE-2026-45584.
  • Analysis of the 'link following' mechanism in Defender to identify similar patterns in other Windows security components.
  • Further disclosures from Chaotic Eclipse or other researchers regarding additional Defender flaws.
  • Adoption of the patches in enterprise environments and any reports of bypasses.
  • CISA KEV updates for related flaws in other endpoint security products (e.g., CrowdStrike, SentinelOne).

背景与对比

这并非Microsoft Defender首次被发现存在漏洞；它有过严重缺陷的历史（例如CVE-2021-1647、CVE-2022-24521）。然而，安全产品中权限提升漏洞的积极利用尤其值得注意，因为它直接削弱了对端点保护的信任。与公开披露的零日漏洞（RedSun、UnDefend）的关联表明，漏洞研究团体越来越多地针对安全软件，并且利用代码可能被威胁行为者迅速武器化。在同一KEV更新中包含数十年前的IE和DirectX漏洞，突显了遗留代码的持续风险，这与Defender的现代攻击面形成了对比。

重要性

这些漏洞之所以重要，是因为它们表明安全软件本身就是一个关键的攻击面。成功利用Defender可以禁用或绕过数百万Windows端点上的主要防御机制，从而实现进一步的入侵。权限提升漏洞（CVE-2026-41091）是通往SYSTEM（Windows上的最高权限级别）的直接路径，这是恶意软件和后利用工具包的关键目标。堆溢出漏洞（CVE-2026-45584）是一个潜在的可蠕虫化的RCE向量，存在于处理网络流量或文件的组件中，使其成为补丁分析和利用开发研究的高优先级目标。

批判性视角

这篇文章在很大程度上依赖于微软的公告和研究人员的主张。CVE-2026-41091和CVE-2026-45498的确切利用细节（例如攻击向量、所需的用户交互）并未完全描述。与“RedSun”和“UnDefend”的关联基于重叠的描述，而非微软的确认。拒绝服务漏洞（CVE-2026-45498）的CVSS评分较低（4.0），表明其影响可能有限，但其被纳入KEV目录表明存在实际的武器化。对于默认配置，声称无需采取任何操作即可更新是正确的，但企业管理的系统可能有延迟更新策略。
