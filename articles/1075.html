1) Why-read

PPAI is the first system for personalized LLM agent interoperability in P2P edge networks, achieving up to 7.96% accuracy improvement and 16.34% latency reduction by matching queries to specialized remote agents.

2) CRGP

C — Context

Deploying LLMs on edge devices enables personalized agents via fine-tuning or RAG, but individual agents are limited to their owner's domain. Existing multi-agent collaboration systems assume co-located models on a single device, not decentralized P2P settings with dynamic agent pools.

R — Related work

  • RouterDC and KABB perform agent routing but assume on-device co-location
  • Mobile-Agent-v2 and MobileSteward focus on single-device mobile agent orchestration
  • P2P file-sharing systems (BitTorrent, DHT) handle uniform data, not specialized agent competencies
  • Multi-agent frameworks (MetaGPT, SWE-agent) enable collaboration but not in decentralized edge environments

G — Gap

No existing system supports query-agent matching and load balancing in a fully decentralized P2P network where agents have diverse specializations, churn dynamically, and real-time load is unobservable. Prior routing methods cannot handle the scalability and interchangeability challenges of such environments.

P — Proposal

PPAI introduces a prototype-based query-agent pair scoring mechanism that projects queries and agents into a fixed latent space for efficient matching under churn. It also formulates routing as a Bayesian game with a Cost of Delegation metric, using belief distributions from historical interactions to estimate peer load and achieve Bayesian Nash equilibrium.

4) Experiments

Main results

On multiple tasks, PPAI achieves average accuracy improvement up to 7.96% and reduces latency by 16.34% compared to baseline under high user demand simulation.

Ablation

The prototype-based scoring mechanism and Bayesian game load balancing both contribute significantly; removing either component degrades accuracy and increases latency, though exact ablation numbers are not provided in the abstract.

Limitations

['The system assumes each user deploys only one specialized agent locally, limiting multi-agent scenarios on a single device', 'Belief-based load estimation may lag under extreme churn or bursty query arrivals', 'Evaluation is simulation-based; real-world deployment on heterogeneous edge hardware is not demonstrated']

5) Why it matters

  1. Enables collaborative edge intelligence without central servers, aligning with privacy-preserving and low-latency requirements
  2. Prototype-based scoring is a novel approach for dynamic agent matching that could generalize to other P2P AI systems
  3. Bayesian game formulation provides theoretical grounding for decentralized load balancing under partial observability

6) Next steps

  • [ ] Implement PPAI on real edge devices (e.g., AI PCs, smartphones) to validate latency and accuracy in practice
  • [ ] Extend the prototype space to support multi-modal queries and agents
  • [ ] Explore incentive mechanisms to encourage agents to share load fairly in the P2P network
