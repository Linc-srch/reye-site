9-Year-Old Linux Kernel Flaw Enables Root Command Execution on Major Distros

9-Year-Old Linux Kernel Flaw Enables Root Command Execution on Major Distros

supply-chain-attack
启示 · Insight

For a kernel security researcher, this is a textbook case of a latent privilege-escalation bug in core kernel logic (ptrace access control) that survived for nearly a decade. The fact that Qualys developed four distinct exploit paths (chage, ssh-keysign, pkexec, accounts-daemon) demonstrates the primitive's robustness and the systemic risk of such a flaw. The disclosure pattern—PoC released shortly after a public kernel commit—highlights the tension between open development and responsible disclosure. The concurrent release of the 'PinTheft' RDS/io_uring exploit further underscores the current high-volume of Linux kernel LPE research.

对于内核安全研究人员来说，这是一个教科书式的案例：一个潜伏了近十年的核心内核逻辑（ptrace访问控制）中的权限提升漏洞。Qualys开发了四种不同的利用路径（chage、ssh-keysign、pkexec、accounts-daemon），这一事实证明了该原语的健壮性以及此类漏洞的系统性风险。披露模式——在公开内核提交后不久发布PoC——凸显了开放开发与负责任披露之间的紧张关系。同时发布的'PinTheft' RDS/io_uring漏洞进一步凸显了当前Linux内核LPE研究的高产状态。

Summary

A 9-year-old Linux kernel vulnerability (CVE-2026-46333, CVSS 5.5) in the __ptrace_may_access() function allows unprivileged local users to execute arbitrary commands as root on major distributions like Debian, Fedora, and Ubuntu.

摘要

一个存在9年的Linux内核漏洞（CVE-2026-46333，CVSS 5.5）位于__ptrace_may_access()函数中，允许无特权的本地用户在Debian、Fedora和Ubuntu等主要发行版上以root身份执行任意命令。

Deep Note

Key Points

  • CVE-2026-46333 is a privilege management flaw in the Linux kernel's __ptrace_may_access() function, introduced in November 2016.
  • It allows an unprivileged local user to disclose sensitive files (/etc/shadow, SSH host keys) and execute arbitrary commands as root.
  • Qualys released four distinct exploits targeting chage, ssh-keysign, pkexec, and accounts-daemon.
  • A PoC was released shortly after a public kernel commit, indicating a fast exploit-to-patch cycle.
  • Temporary workaround: set kernel.yama.ptrace_scope to 2.
  • The article also covers a separate LPE exploit called 'PinTheft' for Arch Linux, exploiting a double-free in RDS zerocopy via io_uring.

Context & Contrast

This vulnerability stands out not for its complexity but for its longevity (9 years) and the breadth of its impact (multiple major distros). It follows a recent string of Linux kernel LPE bugs (Copy Fail, Dirty Frag, Fragnesia), indicating a sustained and productive period for kernel vulnerability research. Unlike many memory-corruption bugs, this is a logic flaw in privilege management, making it a different class of kernel security issue. The 'PinTheft' exploit is notable for chaining an RDS double-free with io_uring, a relatively new and complex attack surface.

Why It Matters

This vulnerability is significant because it provides a reliable, local-to-root privilege escalation on default installations of the most widely used Linux distributions. The ability to read /etc/shadow and SSH host keys compromises the entire system's identity and trust model. The existence of four different exploit paths makes it harder to mitigate by simply patching one user-space component. For the kernel security community, it reinforces the need for rigorous auditing of core kernel functions like ptrace, especially those that have been stable for years.

Critical Take

The CVSS score of 5.5 (Medium) may understate the real-world risk, as it requires local access but then provides a reliable root escalation. The article does not detail the exact conditions required for exploitation (e.g., specific kernel configs, namespaces). The 'PinTheft' exploit has specific prerequisites (RDS module loaded, io_uring enabled, SUID binary, x86_64) which limit its general applicability. The hype around '9-year-old flaw' is valid but should be tempered by the fact that it was undiscovered, not unpatched; the fix is now available.

What to Watch

  • Monitor for mass exploitation attempts in the wild, especially targeting cloud and CI/CD environments where local access is common.
  • Track the adoption of the kernel patch across major distros and the speed of deployment.
  • Watch for further research into __ptrace_may_access() and similar privilege-checking logic in the kernel.
  • Follow the evolution of io_uring-based exploits, as 'PinTheft' demonstrates a new technique for chaining kernel bugs.
  • Observe if the 'ssh-keysign-pwn' codename leads to a broader disclosure campaign or tooling.
深度笔记

要点

  • CVE-2026-46333是Linux内核__ptrace_may_access()函数中的一个权限管理缺陷，于2016年11月引入。
  • 它允许无特权的本地用户泄露敏感文件（/etc/shadow、SSH主机密钥）并以root身份执行任意命令。
  • Qualys发布了四个不同的漏洞利用程序，分别针对chage、ssh-keysign、pkexec和accounts-daemon。
  • 在公开内核提交后不久，PoC即被发布，表明漏洞利用到补丁的周期很快。
  • 临时解决方法：将kernel.yama.ptrace_scope设置为2。
  • 文章还涵盖了另一个针对Arch Linux的名为'PinTheft'的LPE漏洞，该漏洞通过io_uring利用RDS zerocopy中的double-free。

背景与对比

该漏洞的突出之处不在于其复杂性，而在于其存在时间长（9年）和影响范围广（多个主要发行版）。它紧随近期一系列Linux内核LPE漏洞（Copy Fail、Dirty Frag、Fragnesia）之后，表明内核漏洞研究正处于一个持续且高产的时期。与许多内存损坏漏洞不同，这是一个权限管理中的逻辑缺陷，使其成为不同类别的内核安全问题。'PinTheft'漏洞利用程序值得注意，因为它将RDS double-free与io_uring（一个相对较新且复杂的攻击面）链接起来。

重要性

该漏洞意义重大，因为它为最广泛使用的Linux发行版的默认安装提供了一种可靠的、从本地到root的权限提升途径。读取/etc/shadow和SSH主机密钥的能力会破坏整个系统的身份和信任模型。存在四种不同的利用路径，使得仅通过修补一个用户空间组件来缓解漏洞变得更加困难。对于内核安全社区而言，它强化了对核心内核函数（如ptrace）进行严格审计的必要性，尤其是那些多年来一直保持稳定的函数。

批判性视角

CVSS评分5.5（中危）可能低估了实际风险，因为它虽然需要本地访问，但随后能提供可靠的root权限提升。文章没有详细说明利用所需的确切条件（例如，特定的内核配置、命名空间）。'PinTheft'漏洞利用程序有特定的先决条件（加载RDS模块、启用io_uring、SUID二进制文件、x86_64），这限制了其普遍适用性。关于'存在9年的漏洞'的宣传是合理的，但应注意到它之前是未被发现，而非未被修补；现在补丁已经可用。

后续关注

  • 跟踪主要发行版对内核补丁的采用情况和部署速度。
  • 关注针对__ptrace_may_access()及内核中类似权限检查逻辑的进一步研究。
  • 关注基于io_uring的漏洞利用的发展，因为'PinTheft'展示了一种链接内核漏洞的新技术。
