9-Year-Old Linux Kernel Flaw Enables Root Command Execution on Major Distros

9-Year-Old Linux Kernel Flaw Enables Root Command Execution on Major Distros

supply-chain-attack
Summary

Cybersecurity researchers at Qualys have disclosed a nine-year-old Linux kernel vulnerability, CVE-2026-46333, that enables unprivileged local users to execute arbitrary commands as root on major distributions including Debian, Fedora, and Ubuntu. The flaw resides in the __ptrace_may_access() function, introduced in November 2016. Exploitation allows disclosure of sensitive files like /etc/shadow and SSH private keys, as well as root execution via chage, ssh-keysign, pkexec, or accounts-daemon. A proof-of-concept exploit has been released. The article also covers a separate local privilege escalation vulnerability called PinTheft affecting Arch Linux. Users are advised to apply kernel updates immediately.
