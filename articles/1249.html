Cisco Patches CVSS 10.0 Secure Workload REST API Flaw Enabling Data Access

Cisco Patches CVSS 10.0 Secure Workload REST API Flaw Enabling Data Access

supply-chain-attack
Summary

Cisco has released updates to address a maximum-severity vulnerability (CVE-2026-20223, CVSS 10.0) in its Secure Workload product. The flaw stems from insufficient validation and authentication in REST API endpoints, allowing an unauthenticated remote attacker to read sensitive data and make configuration changes across tenant boundaries with Site Admin privileges. The issue affects both SaaS and on-prem deployments. Cisco discovered the flaw during internal testing and reports no evidence of exploitation in the wild. Patches are available for affected releases. This disclosure follows another actively exploited Cisco flaw in Catalyst SD-WAN Controller.

摘要

思科发布了补丁，修复了Secure Workload中的一个最高严重性漏洞（CVE-2026-20223，CVSS 10.0）。该漏洞源于REST API端点的验证和认证不足，未认证的远程攻击者可跨租户边界读取敏感数据并以Site Admin权限进行配置更改。影响SaaS和本地部署。思科在内部测试中发现此漏洞，尚未发现野外利用。补丁已发布。此前思科还披露了Catalyst SD-WAN Controller中另一个被积极利用的漏洞。
