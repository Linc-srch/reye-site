Spreadsheet-RL: Advancing Large Language Model Agents on Realistic Spreadsheet Tasks via Reinforcement Learning

Spreadsheet-RL：通过强化学习推进大型语言模型在真实电子表格任务中的智能体能力

Banghao Chi, Yining Xie, Mingyuan Wu, Jingcheng Yang, Jize Jiang, Zhaoheng Li, Shengyi Qian, Minjia Zhang, Klara Nahrstedt, Rui Hou, Xiangjun Fan, Hanchao Yu

Summary

Spreadsheet-RL demonstrates that on-policy RL with outcome-based rewards can significantly improve LLM agents on complex, multi-step spreadsheet tasks, outperforming prompt-based methods.

摘要

本文提出Spreadsheet-RL框架，通过在线强化学习（GRPO）和基于结果的奖励，显著提升了LLM在复杂多步电子表格任务中的表现。关键创新在于自动化收集配对的起始-目标电子表格数据，并构建了包含Python沙箱和工具路由规则的逼真多轮交互环境。实验表明，Qwen3-4B模型在SpreadsheetBench上的Pass@1从12.0%提升至23.4%，在金融和供应链领域也展现了良好的泛化能力。

Deep Note

Deep Note — spreadsheet-rl

Key Takeaways - RL fine-tuning (GRPO) on spreadsheet tasks improves Qwen3-4B-Thinking-2507 Pass@1 from 12.0% to 23.4% on SpreadsheetBench. - Automated pipeline collects paired start-goal spreadsheets from online forums for scalable RL training data. - Spreadsheet Gym provides a realistic multi-turn Excel environment with Python sandbox and tool-routing rules. - Domain-Spreadsheet benchmark (finance, supply chain) shows generalization: Pass@1 from 8.4% to 17.2%. - Open-source release of data, environment, harness, and training pipeline enables reproducible research.

1) Why-read

Spreadsheet-RL demonstrates that on-policy RL with outcome-based rewards can significantly improve LLM agents on complex, multi-step spreadsheet tasks, outperforming prompt-based methods.

2) CRGP

C — Context

Spreadsheet agents have relied on prompting general-purpose LLMs (e.g., GPT-4o) for simple operations, but struggle with multi-step workflows. Recent RL fine-tuning (e.g., DeepSeek-R1) has improved reasoning in math, code, and web tasks, but applying RL to spreadsheets is challenging due to the difficulty of collecting paired start-goal spreadsheets and the lack of step-by-step supervision.

R — Related work

  • SheetCopilot and SheetAgent use prompt-based agents with GPT-4o for spreadsheet tasks, limited to simple operations.
  • SpreadsheetBench provides 912 paired spreadsheets from forums, but no RL training pipeline.
  • DeepSeek-R1 and GRPO show RL improves reasoning in math and code; applied here to spreadsheet domain.
  • OpenAI Agent uses proprietary internal spreadsheet datasets for evaluation, not open-source.

G — Gap

Existing spreadsheet agents rely on prompting general LLMs and lack model-side training for complex multi-step workflows. No open-source framework provides scalable data collection, realistic environment, and RL fine-tuning for spreadsheet automation.

P — Proposal

Spreadsheet-RL introduces an automated pipeline to collect paired start-goal spreadsheets from online forums, a Spreadsheet Gym environment with Python sandbox and tool-routing rules, and uses GRPO to fine-tune Qwen3 models on outcome-based rewards. The key insight is that RL with verifiable final spreadsheet rewards can improve agentic behavior without step-by-step supervision.

4) Experiments

Main results

On SpreadsheetBench, Qwen3-4B-Thinking-2507 Pass@1 improves from 12.0% to 23.4% after Spreadsheet-RL. On Domain-Spreadsheet (finance, supply chain), Pass@1 rises from 8.4% to 17.2%. GPT-4o achieves 45.5% on SpreadsheetBench as reference.

Ablation

Ablation shows that each component (native harness, full tools, RL training) contributes: base model 12.0%, +harness ~15%, +tools ~18%, +RL 23.4% on SpreadsheetBench.

Limitations

['RL training requires large-scale paired spreadsheet data; automated collection may introduce noise.', 'Evaluation limited to Excel environment; generalization to Google Sheets or LibreOffice not tested.', 'Performance still below GPT-4o on SpreadsheetBench (23.4% vs 45.5%), indicating room for improvement.']

5) Why it matters

  1. Demonstrates that RL with outcome rewards can train specialized agents for data-centric workflows, relevant to your work on automating data pipelines.
  2. Open-source release enables you to adapt the pipeline for custom spreadsheet tasks in finance or supply chain.
  3. Shows that small open-source models (4B) can be competitive with larger models when fine-tuned with RL, reducing dependency on proprietary APIs.

深度笔记

Spreadsheet-RL：通过强化学习推进大型语言模型在真实电子表格任务中的智能体能力

主要结论 - RL微调（GRPO）在电子表格任务上使Qwen3-4B-Thinking-2507的Pass@1从12.0%提升至23.4%。 - 自动化流水线从在线论坛收集配对的起始-目标电子表格，为可扩展的RL训练提供数据。 - Spreadsheet Gym提供了逼真的多轮Excel环境，包含Python沙箱和工具路由规则。 - 在金融和供应链领域的Domain-Spreadbench上，Pass@1从8.4%提升至17.2%，展现了泛化能力。 - 开源了数据、环境、测试框架和训练流水线，支持可重复研究。

1) 阅读理由

核心主张：在线策略RL结合基于结果的奖励能显著提升LLM在复杂多步电子表格任务上的表现；关键观察：无需逐步监督，仅凭最终电子表格的可验证奖励即可改善智能体行为。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：现有电子表格智能体依赖对通用LLM（如GPT-4o）进行提示，仅能处理简单操作，难以应对多步工作流。近期RL微调（如DeepSeek-R1）在数学、代码和网页任务上提升了推理能力，但将RL应用于电子表格面临挑战：难以收集配对的起始-目标电子表格，且缺乏逐步监督。

相关工作：SheetCopilot和SheetAgent使用基于提示的智能体配合GPT-4o，局限于简单操作；SpreadsheetBench提供了912个来自论坛的配对电子表格，但无RL训练流水线；DeepSeek-R1和GRPO展示了RL在数学和代码推理上的提升，本文将其应用于电子表格领域；OpenAI Agent使用专有内部电子表格数据集进行评估，未开源。

差距：现有电子表格智能体依赖提示通用LLM，缺乏针对复杂多步工作流的模型侧训练。没有开源框架提供可扩展的数据收集、逼真的环境和RL微调用于电子表格自动化。

方案：Spreadsheet-RL引入自动化流水线从在线论坛收集配对的起始-目标电子表格，构建了包含Python沙箱和工具路由规则的Spreadsheet Gym环境，并使用GRPO基于结果奖励微调Qwen3模型。关键洞察是：RL结合可验证的最终电子表格奖励，无需逐步监督即可改善智能体行为。

3) 实验

在SpreadsheetBench上，Qwen3-4B-Thinking-2507的Pass@1从12.0%提升至23.4%。在Domain-Spreadsheet（金融和供应链）上，Pass@1从8.4%提升至17.2%。作为参考，GPT-4o在SpreadsheetBench上达到45.5%。

消融实验

消融实验表明每个组件均有贡献：基础模型12.0%，+原生测试框架约15%，+完整工具约18%，+RL训练达到23.4%（在SpreadsheetBench上）。

局限性

RL训练需要大规模配对的电子表格数据，自动化收集可能引入噪声；评估仅限于Excel环境，未测试对Google Sheets或LibreOffice的泛化性；性能仍低于GPT-4o（23.4% vs 45.5%），表明有改进空间。

4) 重要性

  1. 展示了RL结合结果奖励可以训练面向数据密集型工作流的专用智能体，对自动化数据流水线相关工作有直接参考价值。
  2. 开源发布使你可以将流水线适配到金融或供应链领域的自定义电子表格任务。
  3. 表明小型开源模型（4B）通过RL微调可以与大模型竞争，减少对专有API的依赖。
