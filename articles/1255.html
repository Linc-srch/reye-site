GitHub Internal Repositories Breached via Malicious Nx Console VS Code Extension

GitHub Internal Repositories Breached via Malicious Nx Console VS Code Extension

supply-chain-attack Malware & Threats Supply Chain Security
Summary

GitHub confirmed that a breach of its internal repositories was caused by a malicious version of the Nx Console VS Code extension, which was deployed after an attacker compromised a developer's system. The tampered extension was live on the Visual Studio Marketplace for only 18 minutes but installed a credential stealer that harvested sensitive data from password managers and cloud services. The attack, attributed to the TeamPCP group, exfiltrated approximately 3,800 repositories. GitHub stated that customer data may have been exposed in limited form and rotated critical secrets. The incident underscores the dangers of auto-updating extension marketplaces and the interconnected nature of software supply chain attacks.

摘要

GitHub 确认其内部仓库被入侵源于一个恶意的 Nx Console VS Code 扩展，该扩展在攻击者攻陷开发者系统后被发布。篡改后的扩展仅在 Visual Studio Marketplace 上存活了 18 分钟，但植入了凭据窃取器，能收集来自密码管理器、AWS 和 GitHub 等服务的敏感数据。攻击者 TeamPCP 组织窃取了约 3800 个仓库。GitHub 表示客户数据可能以有限形式暴露，并轮换了关键密钥。此事件凸显了扩展市场自动更新机制的风险及软件供应链安全的连锁效应。
