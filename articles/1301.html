Causal methods for LLM development and evaluation

Deep Note — 因果方法与LLM开发

content-management digital-publishing rlhf AI & Machine Learning

Dennis Frauen, Marie Brockschmidt, Konstantin Hess, Haorui Ma, Yuchen Ma, Abdurahman Maarouf, Maresa Schröder, Jonas Schweisthal, Yuxin Wang, Athiya Deviyani, Sonali Parbhoo, Rahul G. Krishnan, Stefan Feuerriegel

Summary

This paper argues that many central LLM development and evaluation decisions are causal problems, and shows how causal inference methods can address confounding, bias, and non-stationarity in logged data to enable more reliable system design.

摘要

本文系统论证了LLM开发中的关键决策（如数据混合、路由、对齐）本质上是因果问题，并展示了如何利用因果推断方法（如双机器学习、策略学习）从观测数据中估计反事实结果。对于希望提升LLM系统可靠性和评估有效性的研究者，本文提供了统一的理论框架和实用工具。

Deep Note

Deep Note — causal-llm-dev

Key Takeaways - LLM development decisions (data mixtures, routing, alignment) are inherently causal questions about intervention effects. - Logged observational data in LLM pipelines suffer from confounding, selection bias, and distribution shift, making causal methods necessary. - Causal inference provides principled tools (e.g., double machine learning, policy learning) for estimating counterfactual outcomes from non-experimental data. - The paper maps causal opportunities across pretraining, alignment, routing, agentic workflows, and evaluation (e.g., LLM-as-judge). - Causal methods can improve data efficiency, robustness, and statistical validity in LLM development.

1) Why-read

2) CRGP

C — Context

LLM development currently relies on large-scale empirical iteration over data mixtures, reward models, routing strategies, and evaluation pipelines, often using logged observational data. These data are subject to selection bias, distribution shift, and noisy labels, making purely predictive approaches fragile. Causal inference offers a principled framework for estimating intervention effects from such data.

R — Related work

  • Causal machine learning overviews (Kaddour et al., 2025; Frauen et al., 2026b)
  • Causal reasoning in LLMs (Jin et al., 2023, 2024; Kıcıman et al., 2023; Zečević et al., 2023)
  • Causal methods for LLM routing (Tsiourvas et al., 2025)
  • Causal approaches in pretraining and alignment (Frauen et al., 2026a; Bou et al., 2026; Joselowitz et al., 2025; Guerdan et al., 2026; Kobalczyk and van der Schaar, 2025)
  • Double machine learning (Chernozhukov et al., 2018)
  • Policy learning (Kennedy, 2023a,b)

G — Gap

While causal reasoning in LLMs has been studied (LLMs as causal reasoners), the use of causal inference methods as a tool to improve LLM development and evaluation itself is underexplored. Existing LLM pipelines rely heavily on logged data with confounding and distribution shifts, yet developers rarely apply principled causal identification and estimation techniques.

P — Proposal

The paper proposes a unifying causal perspective on LLM development, framing decisions (data mixtures, routing, alignment, evaluation) as interventions with counterfactual outcomes. It maps opportunities for causal methods across the entire pipeline—pretraining, alignment, routing, agentic workflows, and evaluation—and advocates for using techniques like double machine learning and policy learning to estimate causal effects from observational logs.

Deep Note — 因果方法与LLM开发

主要结论 - LLM开发决策（数据混合、路由、对齐）本质上是关于干预效果的因果问题。 - LLM流水线中的观测数据存在混杂、选择偏差和分布偏移，必须使用因果方法。 - 因果推断提供了从非实验数据估计反事实结果的原则性工具（如双机器学习、策略学习）。 - 论文将因果机会映射到预训练、对齐、路由、智能体工作流和评估（如LLM-as-judge）等环节。 - 因果方法可提升LLM开发的数据效率、鲁棒性和统计有效性。

1) 阅读理由

本文的核心主张是：LLM开发和评估中的许多关键决策本质上是因果问题，而因果推断方法能够有效处理观测数据中的混杂、偏差和非平稳性，从而实现更可靠的系统设计。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：当前LLM开发依赖大规模经验迭代（数据混合、奖励模型、路由策略、评估流水线），常用观测数据，但存在选择偏差、分布偏移和噪声标签，纯预测方法脆弱。因果推断为从这类数据估计干预效应提供了原则性框架。

相关工作：涵盖因果机器学习综述、LLM因果推理、LLM路由的因果方法、预训练和对齐中的因果方法、双机器学习、策略学习等。

差距：虽然LLM作为因果推理者已被研究，但将因果推断方法作为改进LLM开发与评估本身的工具仍未被充分探索。现有LLM流水线严重依赖存在混杂和分布偏移的观测数据，开发者很少应用原则性的因果识别与估计技术。

提议：本文提出统一的因果视角，将LLM开发决策（数据混合、路由、对齐、评估）视为具有反事实结果的干预，并映射因果方法在整个流水线（预训练、对齐、路由、智能体工作流、评估）中的机会，倡导使用双机器学习和策略学习等技术从观测日志中估计因果效应。
