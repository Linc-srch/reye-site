Summary

EnvFactory proposes a fully automated framework for scaling tool-use agents by autonomously constructing verified executable environments from authentic online resources and synthesizing natural multi-turn trajectories with implicit intents, achieving up to +15% on BFCLv3 with only 85 environments.

摘要

本文提出EnvFactory，一个全自动框架，通过从真实在线资源自主构建可验证的可执行环境，并合成包含隐式意图的自然多轮轨迹，解决了工具使用智能体在强化学习训练中缺乏可扩展环境和真实数据的问题。仅用85个环境即可在BFCLv3上提升高达15%，数据效率显著优于此前需要5倍以上环境的工作。

1) Why-read

EnvFactory introduces a fully automated pipeline for scalable, executable environment synthesis and robust RL training, achieving state-of-the-art tool-use agent performance with significantly fewer environments.

2) CRGP

C — Context

Tool-use agents powered by LLMs require realistic, executable environments and high-quality training trajectories for Agentic RL. Existing approaches rely on costly real-world APIs, LLM-based simulators that hallucinate, or pre-collected documents, limiting scalability and robustness. Synthetic trajectories often lack the implicit reasoning and natural multi-turn structure found in human interactions.

R — Related work

  • Prior work on tool-use agents uses real API calls (expensive and rate-limited) or LLM-simulated environments (prone to hallucination and limited statefulness).
  • Synthetic environment construction often depends on pre-collected documents or single-turn interactions, failing to capture multi-turn dynamics.
  • Trajectory synthesis methods typically produce over-specified instruction-like sequences rather than natural human intents, reducing RL training effectiveness.
  • Agentic RL frameworks like AgentOhana and ToolLLM rely on large numbers of environments (e.g., 400+) but suffer from data scarcity and quality issues.

G — Gap

Existing methods for scaling tool-use agents face two bottlenecks: lack of scalable, robust execution environments that are stateful and multi-turn, and scarcity of realistic training data that captures implicit human reasoning. Synthetic trajectories are often over-specified and resemble instruction sequences rather than natural intents, reducing their effectiveness for RL training.

P — Proposal

EnvFactory is a fully automated framework with three collaborating agents: a Search Agent that autonomously proposes and searches for authentic sources, a Code Agent that implements databases and code using feedback from a Test Agent, and the Test Agent that generates test cases and error reports. It also introduces QueryGen, which uses topology-aware sampling and calibrated refinement to synthesize natural multi-turn trajectories with implicit intents from the constructed environments.

---

4) Experiments

Main results

Using only 85 verified environments across 7 domains, EnvFactory generates 2,575 SFT and RL trajectories. It improves Qwen3-1.7B by +15%, Qwen3-4B by +12%, and Qwen3-8B by +10% on BFCLv3 multi-turn average. On MCP-Atlas, it achieves +8.6% improvement. On conversational benchmarks, it achieves +6% on τ²-Bench and VitaBench. Resource efficiency analysis shows that with only 85 environments, EnvFactory outperforms prior work using 400+ environments.

Ablation

Ablation on BFCLv3 under different trajectory reward weights shows that a balanced weight (e.g., 0.5) yields the best multi-turn average performance, while extreme weights (0.0 or 1.0) degrade results, confirming the importance of calibrated refinement in trajectory synthesis.

Limitations

['The framework currently focuses on 7 domains; generalization to more diverse or specialized domains may require additional engineering.', 'The quality of synthesized environments depends on the authenticity of the sources found by the Search Agent, which may vary.', 'The approach assumes that topology-aware sampling and calibrated refinement sufficiently capture human-like implicit intents, which may not hold for all task types.']

5) Why it matters

  1. EnvFactory's fully automated pipeline removes the manual bottleneck of environment construction, making Agentic RL scalable and reproducible.
  2. The finding that fewer, higher-quality environments outperform larger, noisier sets is directly actionable for researchers designing training data pipelines.
  3. The topology-aware sampling strategy provides a principled method for generating multi-turn trajectories with implicit intents, addressing a key weakness in prior synthetic data approaches.
  4. The open-source release of environments and trajectories enables community-driven expansion and benchmarking.

深度笔记

EnvFactory：通过可执行环境合成与鲁棒强化学习扩展工具使用智能体

主要结论 - EnvFactory通过搜索、代码、测试三个智能体自主构建可执行环境，无需人工标注或预收集文档。 - 拓扑感知采样递归解析工具依赖关系，生成结构合理的多轮任务。 - 校准精炼阶段向查询注入隐式意图和歧义，使合成轨迹更接近真实人类交互。 - 仅85个环境即可在BFCLv3、MCP-Atlas等基准上取得显著提升，数据效率远超此前方法。

1) 阅读理由

EnvFactory提出全自动环境合成与隐式意图轨迹生成框架，仅用85个环境即可在BFCLv3上提升15%，解决了工具使用智能体RL训练中环境稀缺与数据不真实的关键瓶颈。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

上下文：现有方法依赖昂贵的真实API、易幻觉的LLM模拟器或需预收集文档的合成环境，且合成轨迹常过于显式，缺乏人类隐式推理。相关工作包括生产环境（ToolLLM）、模拟环境（SimulatingEnvironments）、合成环境（AutoForge）和依赖工具图（GTool）。差距：现有合成环境方法依赖预收集文档或现有任务集，泛化受限；合成轨迹过于显式，不利于RL训练。提案：EnvFactory通过三智能体流水线（搜索、代码、测试）自主提出工具使用场景、探索在线资源并构建可执行环境，引入拓扑感知采样和校准精炼阶段，生成自然多轮轨迹。

3) 实验

使用85个已验证环境（覆盖7个领域）生成2,575条轨迹（1,622条SFT + 953条RL）。在Qwen3系列模型上，BFCLv3提升高达15%，MCP-Atlas提升8.6%，对话基准（τ²-Bench和VitaBench）提升6%，优于使用5倍以上环境的先前工作。

消融实验

消融实验表明，拓扑感知采样和校准精炼均对性能有贡献；移除任一组件会降低轨迹质量和下游任务准确率（具体数值在论文中显示通过率和自然度指标显著下降）。

局限性

['框架仍需访问真实在线资源来构建环境，可能存在覆盖偏差。', '环境数量（85个）相对较少，未展示扩展到数百或数千领域的能力。', '校准精炼步骤若未针对不同领域仔细调参，可能引入不自然的人为痕迹。']

4) 重要性

  1. 全自动环境构建消除了人工标注，使智能体RL训练可规模化。
  2. 拓扑感知采样与隐式意图注入解决了合成轨迹的真实性差距，提升泛化能力。
  3. 数据效率（85个环境优于5倍以上环境）表明环境和轨迹质量比数量更重要。
