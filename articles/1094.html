Summary

EnvFactory proposes a fully automated framework for scaling tool-use agents by autonomously constructing verified executable environments from authentic online resources and synthesizing natural multi-turn trajectories with implicit intents, achieving up to +15% on BFCLv3 with only 85 environments.

摘要

本文提出EnvFactory，一个全自动框架，通过从真实在线资源自主构建可验证的可执行环境，并合成包含隐式意图的自然多轮轨迹，解决了工具使用智能体在强化学习训练中缺乏可扩展环境和真实数据的问题。仅用85个环境即可在BFCLv3上提升高达15%，数据效率显著优于此前需要5倍以上环境的工作。

1) Why-read

EnvFactory proposes a fully automated pipeline for scalable, executable tool environment synthesis and realistic multi-turn trajectory generation, achieving state-of-the-art agentic RL training efficiency with far fewer environments.

2) CRGP

C — Context

Tool-use agents require robust execution environments and realistic training data for Agentic Reinforcement Learning (Agentic RL). Existing methods rely on costly real-world APIs, hallucination-prone LLM simulators, or single-turn synthetic environments that often produce over-specified trajectories lacking implicit human reasoning.

R — Related work

  • Prior work on tool-use agent training uses real API calls (e.g., ToolBench, API-Bank) which are expensive and limited in scale.
  • LLM-based simulators (e.g., ToolEmu, APIGen) hallucinate execution results and lack statefulness.
  • Synthetic environment approaches (e.g., AppWorld, AndroidWorld) are often single-turn or depend on pre-collected documents, limiting multi-turn interaction realism.
  • Agentic RL methods (e.g., DPO, GRPO) for tool-use agents suffer from data scarcity and unrealistic trajectory distributions.

G — Gap

Existing approaches for training tool-use agents either depend on costly real-world APIs, hallucination-prone LLM simulators, or synthetic environments that are single-turn and produce over-specified instruction-like trajectories. There is no fully automated framework that can both construct verified, stateful, multi-turn environments and synthesize natural trajectories with implicit human intents at scale.

P — Proposal

EnvFactory is a fully automated framework with three collaborating agents: a Search Agent that autonomously finds authentic tool resources, a Code Agent that implements executable environments, and a Test Agent that verifies correctness. For trajectory synthesis, it uses topology-aware sampling to generate diverse multi-turn paths and calibrated refinement to convert over-specified instructions into natural queries with implicit intents.

---

4) Experiments

Main results

EnvFactory uses only 85 verified environments across 7 domains to generate 2,575 SFT and RL trajectories. It improves Qwen3-1.7B, Qwen3-4B, and Qwen3-8B by up to +15% on BFCLv3 multi-turn average, +8.6% on MCP-Atlas, and +6% on τ²-Bench and VitaBench conversational benchmarks, using 5x fewer environments than prior work.

Ablation

Ablation on BFCLv3 under different trajectory reward weights shows that the calibrated refinement step (converting over-specified instructions to implicit intents) is critical for RL training effectiveness, with the best configuration achieving the reported +15% improvement.

Limitations

['The framework currently focuses on 7 domains; scaling to more diverse or niche domains may require additional verification effort.', "The quality of synthesized environments depends on the Search Agent's ability to find authentic, up-to-date resources, which may vary across domains.", 'The trajectory synthesis assumes a fixed topology; highly dynamic or user-driven interaction patterns may not be fully captured.']

5) Why it matters

  1. Provides a scalable, fully automated pipeline for generating training data for tool-use agents, directly addressing a key bottleneck in agentic RL.
  2. Demonstrates that quality over quantity (85 environments vs. 400+) yields better training efficiency and downstream performance, which is a practical insight for resource-constrained research.
  3. The topology-aware sampling and implicit intent refinement techniques are transferable to other agent training paradigms beyond tool-use.
  4. Offers a robust alternative to costly real-world APIs and hallucination-prone simulators, making agentic RL more accessible.

6) Next steps

  • [ ] Apply EnvFactory to additional domains (e.g., healthcare, finance, scientific computing) to test scalability and domain transfer.
  • [ ] Integrate the synthesized environments and trajectories into existing agentic RL pipelines (e.g., GRPO, DPO) for further validation.
  • [ ] Explore dynamic topology adaptation to handle user-driven or non-linear interaction patterns in trajectory synthesis.

深度笔记

EnvFactory：通过可执行环境合成与鲁棒强化学习扩展工具使用智能体

主要结论 - EnvFactory通过搜索、代码、测试三个智能体自主构建可执行环境，无需人工标注或预收集文档。 - 拓扑感知采样递归解析工具依赖关系，生成结构合理的多轮任务。 - 校准精炼阶段向查询注入隐式意图和歧义，使合成轨迹更接近真实人类交互。 - 仅85个环境即可在BFCLv3、MCP-Atlas等基准上取得显著提升，数据效率远超此前方法。

1) 阅读理由

EnvFactory提出全自动环境合成与隐式意图轨迹生成框架，仅用85个环境即可在BFCLv3上提升15%，解决了工具使用智能体RL训练中环境稀缺与数据不真实的关键瓶颈。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

上下文：现有方法依赖昂贵的真实API、易幻觉的LLM模拟器或需预收集文档的合成环境，且合成轨迹常过于显式，缺乏人类隐式推理。相关工作包括生产环境（ToolLLM）、模拟环境（SimulatingEnvironments）、合成环境（AutoForge）和依赖工具图（GTool）。差距：现有合成环境方法依赖预收集文档或现有任务集，泛化受限；合成轨迹过于显式，不利于RL训练。提案：EnvFactory通过三智能体流水线（搜索、代码、测试）自主提出工具使用场景、探索在线资源并构建可执行环境，引入拓扑感知采样和校准精炼阶段，生成自然多轮轨迹。

3) 实验

使用85个已验证环境（覆盖7个领域）生成2,575条轨迹（1,622条SFT + 953条RL）。在Qwen3系列模型上，BFCLv3提升高达15%，MCP-Atlas提升8.6%，对话基准（τ²-Bench和VitaBench）提升6%，优于使用5倍以上环境的先前工作。

消融实验

消融实验表明，拓扑感知采样和校准精炼均对性能有贡献；移除任一组件会降低轨迹质量和下游任务准确率（具体数值在论文中显示通过率和自然度指标显著下降）。

局限性

['框架仍需访问真实在线资源来构建环境，可能存在覆盖偏差。', '环境数量（85个）相对较少，未展示扩展到数百或数千领域的能力。', '校准精炼步骤若未针对不同领域仔细调参，可能引入不自然的人为痕迹。']

4) 重要性

  1. 全自动环境构建消除了人工标注，使智能体RL训练可规模化。
  2. 拓扑感知采样与隐式意图注入解决了合成轨迹的真实性差距，提升泛化能力。
  3. 数据效率（85个环境优于5倍以上环境）表明环境和轨迹质量比数量更重要。
