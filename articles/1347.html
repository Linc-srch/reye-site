JINX-0164 Targets Cryptocurrency Firms with Fake Recruiter Lures and macOS Malware

JINX-0164 Targets Cryptocurrency Firms with Fake Recruiter Lures and macOS Malware

supply-chain-attack social-engineering
启示 · Insight

For a system security researcher focused on OS/kernel security and software supply-chain attacks, this campaign demonstrates a sophisticated cross-platform attack chain that moves from macOS endpoint compromise (via social engineering) to lateral movement into internal CI/CD systems and source-code repositories. The use of macOS-specific malware (AUDIOFIX) that masquerades as a system audio driver (coreaudiod) and persists via launchctl highlights the growing threat surface on macOS, particularly in developer environments. The supply-chain angle—compromising an npm package (@velora-dex/sdk) to deliver MiniRAT—underscores the need for rigorous dependency integrity checks and build-system hardening. The attacker's ability to modify source code to steal cryptocurrency wallet credentials from other endpoints is a direct threat to software supply-chain security.

对于专注于操作系统/内核安全和软件供应链安全的研究人员而言，此攻击活动展示了一条复杂的跨平台攻击链：从macOS端点入侵（通过社会工程学）横向移动到内部CI/CD系统和源代码仓库。macOS恶意软件AUDIOFIX伪装成系统音频驱动（coreaudiod）并通过launchctl持久化，凸显了macOS（尤其是开发者环境）日益增长的攻击面。供应链角度——通过篡改npm包（@velora-dex/sdk）分发MiniRAT——强调了严格的依赖完整性检查和构建系统加固的必要性。攻击者修改源代码以从其他端点窃取加密货币钱包凭证的能力，直接威胁到软件供应链安全。

Summary

A previously undocumented threat actor, JINX-0164, targets cryptocurrency firms using fake recruiter lures on LinkedIn to deploy macOS malware (AUDIOFIX, MiniRAT) for credential theft and supply-chain compromise of CI/CD infrastructure.

摘要

此前未记录的威胁行为者JINX-0164利用LinkedIn上的虚假招聘诱饵针对加密货币公司，部署macOS恶意软件（AUDIOFIX、MiniRAT）以窃取凭证并入侵CI/CD基础设施，实现供应链攻击。

Deep Note

Key Points

  • Threat actor JINX-0164 targets cryptocurrency firms via fake recruiter lures on LinkedIn.
  • Initial compromise: victims download a fake teleconference app that triggers a bash script to fetch AUDIOFIX (Python infostealer/RAT).
  • AUDIOFIX masquerades as system audio driver coreaudiod, saved as ChromeUpdater, executed via launchctl.
  • Lateral movement: AUDIOFIX injects payload into internal CI/CD systems and modifies source code to steal cryptocurrency wallet credentials.
  • Stolen data includes credentials from password managers, browsers, iCloud Keychain, SSH keys, config files, crypto wallet addresses, and active Discord/Slack/Telegram sessions.
  • Second malware: MiniRAT (Go-based backdoor) distributed via compromised npm package @velora-dex/sdk.
  • No confirmed attribution to North Korea despite behavioral similarities to BlueNoroff/Contagious Interview.

Context & Contrast

This campaign is notable for combining macOS-specific malware (AUDIOFIX) with supply-chain compromise (npm package) in a single operation targeting cryptocurrency developers. Previous macOS malware campaigns (e.g., RustBucket, DPRK-linked) often focused on single-stage infostealers or RATs, but JINX-0164 demonstrates multi-stage lateral movement into CI/CD pipelines—a more advanced operational security (OPSEC) pattern. The use of fake recruiter lures is not new (e.g., Contagious Interview), but the integration with npm supply-chain attacks and macOS-specific persistence (launchctl) represents an escalation in sophistication.

Why It Matters

For system security researchers, this campaign highlights the convergence of endpoint compromise (macOS) with software supply-chain attacks (npm). The ability to move from a developer's laptop to CI/CD systems and modify source code represents a critical threat to the integrity of the software supply chain. The use of macOS-specific malware that mimics system drivers (coreaudiod) and persists via launchctl underscores the need for better macOS security monitoring, especially in developer environments. The campaign also demonstrates that social engineering remains a highly effective initial vector, bypassing technical controls.

Critical Take

The article is based on Wiz's threat intelligence, which provides detailed technical analysis but lacks independent verification of the attribution claims. The 'no infrastructure overlaps with North Korean groups' statement is based on Wiz's own tracking, which may be incomplete. The effectiveness of the supply-chain attack (npm package compromise) is not fully detailed—e.g., how the package was compromised (typosquatting, credential theft, or maintainer compromise) is unclear. The scale of the campaign (number of victims, financial losses) is not disclosed. The macOS malware's capabilities are well-documented, but its evasion techniques (e.g., against macOS Gatekeeper, notarization) are not discussed.

What to Watch

  • Follow-up analysis of the compromised npm package (@velora-dex/sdk) to understand the attack vector (typosquatting vs. credential theft).
  • Monitor for similar campaigns using macOS-specific malware targeting CI/CD systems.
  • Track any new CVEs or vulnerability disclosures related to macOS launchctl persistence or driver masquerading.
  • Watch for attribution updates: if infrastructure overlaps with known North Korean groups are found.
  • Observe whether the AUDIOFIX malware is adapted for Windows/Linux targets.

背景与对比

此攻击活动将macOS特定恶意软件（AUDIOFIX）与供应链攻击（npm包）结合在单一行动中，针对加密货币开发者，值得关注。以往的macOS恶意软件活动（如RustBucket、与朝鲜相关的活动）通常侧重于单阶段信息窃取器或RAT，但JINX-0164展示了向CI/CD流水线的多阶段横向移动——这是一种更高级的操作安全模式。虚假招聘诱饵并非新手法（如Contagious Interview），但与npm供应链攻击和macOS特定持久化（launchctl）的结合代表了复杂度的升级。

重要性

对于系统安全研究人员而言，此活动凸显了端点入侵（macOS）与软件供应链攻击（npm）的融合。从开发者笔记本电脑横向移动到CI/CD系统并修改源代码的能力，对软件供应链的完整性构成了关键威胁。使用模仿系统驱动（coreaudiod）并通过launchctl持久化的macOS特定恶意软件，强调了在开发者环境中加强macOS安全监控的必要性。该活动还表明，社会工程学仍然是绕过技术控制的高效初始向量。

批判性视角

文章基于Wiz的威胁情报，提供了详细的技术分析，但缺乏对归属声明的独立验证。'与朝鲜组织无基础设施重叠'的声明基于Wiz自身的追踪，可能不完整。供应链攻击（npm包篡改）的有效性未完全说明——例如，包是如何被篡改的（域名抢注、凭证窃取或维护者入侵）尚不清楚。活动的规模（受害者数量、财务损失）未披露。macOS恶意软件的能力有详细记录，但其规避技术（如针对macOS Gatekeeper、公证）未讨论。

后续关注

  • 监测使用macOS特定恶意软件针对CI/CD系统的类似活动。
  • 追踪与macOS launchctl持久化或驱动伪装相关的新CVE或漏洞披露。
  • 关注归属更新：是否发现与已知朝鲜组织的基础设施重叠。
  • 观察AUDIOFIX恶意软件是否被适配用于Windows/Linux目标。
