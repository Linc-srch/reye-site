Drupal Core SQL Injection Bug Actively Exploited, Added to CISA KEV

Drupal Core SQL Injection Bug Actively Exploited, Added to CISA KEV

supply-chain-attack
启示 · Insight

For a system security researcher focused on OS, kernel, and software safety, this vulnerability is a classic example of a server-side web application flaw (SQL injection) that can lead to privilege escalation and remote code execution. While not a kernel or memory-safety bug, its exploitation pattern—rapid weaponization after patch release, widespread scanning, and targeting of specific database backends (PostgreSQL)—highlights the operational security challenges in the software supply chain and the importance of rapid patch deployment. The fact that it enables privilege escalation and RCE from a database abstraction layer flaw is a reminder that application-level vulnerabilities can directly compromise the underlying system's integrity.

对于专注于操作系统、内核和软件安全性的系统安全研究员来说，此漏洞是一个典型的服务器端 Web 应用程序缺陷（SQL 注入）示例，可导致权限提升和远程代码执行。虽然这不是内核或内存安全错误，但其利用模式——补丁发布后迅速武器化、广泛扫描以及针对特定数据库后端（PostgreSQL）——突显了软件供应链中的运营安全挑战以及快速部署补丁的重要性。该漏洞能够通过数据库抽象层缺陷实现权限提升和 RCE，这提醒我们，应用程序级别的漏洞可以直接危及底层系统的完整性。

Summary

CISA added CVE-2026-9082, a critical SQL injection vulnerability in Drupal Core, to its KEV catalog after observing active exploitation. Over 15,000 attack attempts targeting 6,000 sites have been detected, primarily probing for vulnerable PostgreSQL-backed Drupal instances.

摘要

CISA 将 Drupal Core 中的一个严重 SQL 注入漏洞 CVE-2026-9082 添加到其 KEV 目录，原因是观察到活跃的利用。已检测到超过 15,000 次攻击尝试，针对 6,000 个站点，主要探测存在漏洞的 PostgreSQL 后端 Drupal 实例。

Deep Note

Key Points

  • CVE-2026-9082 is an SQL injection vulnerability in Drupal Core (CVSS 6.5).
  • CISA added it to the Known Exploited Vulnerabilities (KEV) catalog based on active exploitation evidence.
  • Patches were released less than two days before exploitation was confirmed.
  • Imperva observed over 15,000 attack attempts targeting nearly 6,000 sites across 65 countries.
  • Attacks primarily target gaming and financial services sites (50% of attacks).
  • Most observed activity is probing/reconnaissance, targeting PostgreSQL-backed Drupal configurations.
  • Successful exploitation could lead to privilege escalation and remote code execution.
  • FCEB agencies must apply fixes by May 27, 2026.

Context & Contrast

This is a classic SQL injection vulnerability in a widely-used CMS, but its rapid addition to CISA KEV (within days of patch) and the scale of observed scanning (15k attempts) is notable. It contrasts with more complex memory-corruption or kernel-level exploits that require significant sophistication. The targeting of specific database backends (PostgreSQL) is a tactical detail that differentiates it from generic SQLi attacks. The speed of exploitation (patch-to-exploit in <2 days) aligns with the trend of 'N-day' exploitation becoming nearly as dangerous as '0-day'.

Why It Matters

For system security, this matters because it demonstrates that application-layer vulnerabilities (SQLi) can be a direct vector for privilege escalation and RCE, compromising the entire host. The rapid, widespread scanning indicates that attackers are automating the discovery of vulnerable instances, making unpatched systems extremely high-risk. This reinforces the need for robust patch management, vulnerability scanning, and the principle of least privilege at the application and database level. It also highlights the supply-chain risk of using complex CMS frameworks.

Critical Take

The article is based on vendor reports (Imperva, Drupal) and CISA announcements. The exact exploitation method and the final payload (e.g., data exfiltration vs. RCE) are not detailed. The CVSS score of 6.5 is moderate, but the impact (privilege escalation, RCE) suggests the score may be understated or the attack complexity is higher than typical SQLi. The claim of 'active exploitation' is credible but the scale and sophistication of the attacks (mostly probing) should be monitored. The specific PostgreSQL targeting is a key detail that needs further verification from independent sources.

What to Watch

  • Public disclosure of the specific SQL injection vector and proof-of-concept (PoC) code.
  • Reports of successful exploitation leading to data breaches or ransomware deployment on Drupal sites.
  • Analysis of whether the PostgreSQL-specific targeting is due to a quirk in the Drupal database abstraction layer or a broader technique.
  • Emergence of mass exploitation campaigns (e.g., botnets) targeting this CVE.
  • Updates from Drupal on whether older, unsupported versions (8.9, 9.5) are being targeted more heavily.
深度笔记

要点

  • CVE-2026-9082 是 Drupal Core 中的一个 SQL 注入漏洞（CVSS 6.5）。
  • CISA 基于活跃的利用证据将其添加到已知被利用漏洞（KEV）目录。
  • 在确认被利用前不到两天，补丁已发布。
  • Imperva 观察到超过 15,000 次攻击尝试，针对 65 个国家的近 6,000 个站点。
  • 攻击主要针对游戏和金融服务网站（占攻击的 50%）。
  • 大多数观察到的活动是探测/侦察，针对 PostgreSQL 后端的 Drupal 配置。
  • 成功利用可能导致权限提升和远程代码执行。
  • FCEB 机构必须在 2026 年 5 月 27 日前应用修复程序。

背景与对比

这是一个广泛使用的 CMS 中的经典 SQL 注入漏洞，但其在补丁发布后几天内迅速被添加到 CISA KEV，以及观察到的扫描规模（15k 次尝试）值得注意。这与需要更高复杂度的内存损坏或内核级漏洞形成对比。针对特定数据库后端（PostgreSQL）是一个战术细节，使其与通用的 SQLi 攻击区分开来。利用速度（补丁到利用 <2 天）符合 'N-day' 利用变得几乎与 '0-day' 一样危险的趋势。

重要性

对于系统安全而言，这很重要，因为它表明应用层漏洞（SQLi）可以成为权限提升和 RCE 的直接向量，从而危及整个主机。快速、广泛的扫描表明攻击者正在自动化发现易受攻击的实例，这使得未打补丁的系统风险极高。这强化了对稳健的补丁管理、漏洞扫描以及在应用程序和数据库级别遵循最小权限原则的需求。它也凸显了使用复杂 CMS 框架的供应链风险。

批判性视角

文章基于供应商报告（Imperva、Drupal）和 CISA 公告。确切的利用方法和最终载荷（例如，数据窃取 vs. RCE）没有详细说明。CVSS 评分为 6.5，属于中等，但影响（权限提升、RCE）表明评分可能被低估，或者攻击复杂性高于典型的 SQLi。'活跃利用'的说法是可信的，但攻击的规模和复杂性（主要是探测）需要持续监控。针对特定 PostgreSQL 是一个关键细节，需要来自独立来源的进一步验证。

后续关注

  • 特定 SQL 注入向量和概念验证（PoC）代码的公开披露。
  • 成功利用导致 Drupal 站点数据泄露或勒索软件部署的报告。
  • 分析针对特定 PostgreSQL 是由于 Drupal 数据库抽象层的特性还是更广泛的技术。
  • 针对此 CVE 的大规模利用活动（例如，僵尸网络）的出现。
  • Drupal 关于较旧、不受支持的版本（8.9, 9.5）是否成为更严重目标的更新。
