Drupal Core SQL Injection Bug Actively Exploited, Added to CISA KEV

Drupal Core SQL Injection Bug Actively Exploited, Added to CISA KEV

supply-chain-attack
启示 · Insight

For a system security researcher focused on OS & kernel security and vulnerability research, this incident highlights a critical pattern: a database-layer SQL injection (CVE-2026-9082) in a widely-deployed content management system (Drupal) is being weaponized for privilege escalation and potentially remote code execution. The rapid exploitation timeline (patches released, then active exploitation detected within two days) underscores the shrinking window for patch deployment. The attack vector—specially crafted requests via the database abstraction API—is a classic example of how application-layer flaws can lead to system-level compromise, including potential sandbox escape or container breakout if Drupal runs in a containerized environment. The focus on PostgreSQL-backed configurations is a specific operational detail that informs threat modeling for Drupal deployments.

对于专注于操作系统与内核安全以及漏洞研究的系统安全研究员来说，此事件突出了一个关键模式：广泛部署的内容管理系统 (Drupal) 中的数据库层 SQL 注入漏洞 (CVE-2026-9082) 正被武器化，用于权限提升和潜在的远程代码执行。快速的利用时间线（补丁发布后两天内即检测到主动利用）凸显了补丁部署窗口的急剧缩短。攻击向量——通过数据库抽象 API 发送特制请求——是应用层缺陷如何导致系统级入侵的经典案例，如果 Drupal 在容器化环境中运行，则可能导致沙箱逃逸或容器突破。攻击主要针对使用 PostgreSQL 后端的配置，这一具体的操作细节为 Drupal 部署的威胁建模提供了信息。

Summary

CISA has added CVE-2026-9082, a critical SQL injection vulnerability in Drupal Core, to its Known Exploited Vulnerabilities (KEV) catalog following active exploitation. Over 15,000 attack attempts targeting nearly 6,000 sites have been observed, primarily probing PostgreSQL-backed Drupal instances.

摘要

CISA 已将 Drupal Core 中的一个严重 SQL 注入漏洞 CVE-2026-9082 添加到其已知被利用漏洞 (KEV) 目录中，原因是该漏洞已被积极利用。目前已观察到超过 15,000 次攻击尝试，针对近 6,000 个网站，主要探测使用 PostgreSQL 后端的 Drupal 实例。

Deep Note

Key Points

  • CVE-2026-9082 is an SQL injection vulnerability in Drupal Core (CVSS 6.5).
  • CISA added it to the KEV catalog based on evidence of active exploitation.
  • Exploitation was detected within two days of the patch release.
  • Imperva observed over 15,000 attack attempts targeting nearly 6,000 sites across 65 countries.
  • Attacks primarily target gaming and financial services sites (almost 50% of attacks).
  • Most observed activity is probing, specifically targeting PostgreSQL-backed Drupal configurations.
  • Successful exploitation could lead to privilege escalation and remote code execution.
  • Patches are available for multiple Drupal versions (10.x, 11.x, and manual patches for 8.9/9.x).
  • FCEB agencies must apply fixes by May 27, 2026.

Context & Contrast

This event is a textbook example of a high-profile, actively exploited vulnerability in a major open-source CMS, following the pattern of previous Drupalgeddon-style flaws. The key difference is the rapidity of exploitation post-patch (2 days) and the specific targeting of PostgreSQL-backed instances, which is a more granular operational detail than typical broad SQL injection campaigns. The inclusion in CISA KEV is standard for critical flaws with active exploitation, but the scale (15k attempts) and sector targeting (gaming/finance) are notable. Compared to kernel-level SQLi (e.g., in database engines themselves), this is an application-layer flaw, but its potential for privilege escalation and RCE makes it a systems security concern.

Why It Matters

For system security researchers, this vulnerability matters because it demonstrates a clear path from a web application SQL injection to system-level compromise (privilege escalation, RCE). It highlights the importance of defense-in-depth: even if the OS/kernel is hardened, a vulnerable application like Drupal can be the entry point for a full system takeover. The rapid exploitation timeline pressures organizations to have fast, reliable patch management and incident response processes. The specific targeting of PostgreSQL configurations provides a real-world data point for threat modeling and risk assessment of Drupal deployments. The potential for RCE means this could be used to deploy malware, establish persistence, or pivot to other systems, all of which are within the scope of system security.

Critical Take

The article is based on vendor reports (Imperva, Drupal) and CISA announcements. The exact exploitation method, payloads, and post-exploitation behavior are not detailed. The claim of 'privilege escalation and remote code execution' is from CISA's advisory, but the article does not provide independent verification or technical analysis of the exploit chain. The 'probing' nature of most attacks suggests the full exploitation capability may not yet be widely deployed, or attackers are still refining their techniques. The CVSS score of 6.5 is relatively moderate, but the potential impact (RCE) is severe; this discrepancy warrants further technical investigation.

What to Watch

  • Technical analysis or proof-of-concept (PoC) code for CVE-2026-9082 being published.
  • Reports of actual data exfiltration or ransomware deployments linked to this vulnerability.
  • Updates from Drupal or security vendors on the specific exploitation techniques (e.g., how the SQLi leads to RCE).
  • Whether the vulnerability is chained with other flaws (e.g., for container escape or lateral movement).
  • Scanning activity for other Drupal versions or configurations (e.g., MySQL-backed) to see if the attack broadens.
  • Patch adoption rates and any reports of unpatched systems being compromised.

背景与对比

此事件是主要开源 CMS 中一个高调、被积极利用的漏洞的教科书式案例，遵循了之前 Drupalgeddon 风格漏洞的模式。关键区别在于补丁发布后利用的速度（2 天）以及专门针对使用 PostgreSQL 后端的实例，这比典型的广泛 SQL 注入活动提供了更细粒度的操作细节。将其纳入 CISA KEV 对于存在主动利用的严重漏洞来说是标准操作，但其规模（15,000 次尝试）和针对的行业（游戏/金融）值得注意。与内核级别的 SQL 注入（例如，在数据库引擎本身中）相比，这是一个应用层漏洞，但其权限提升和 RCE 的潜力使其成为系统安全问题。

重要性

对于系统安全研究人员来说，此漏洞之所以重要，是因为它展示了从 Web 应用程序 SQL 注入到系统级入侵（权限提升、RCE）的清晰路径。它强调了纵深防御的重要性：即使操作系统/内核是安全的，像 Drupal 这样存在漏洞的应用程序也可能成为完全接管系统的入口点。快速的利用时间线给组织带来了压力，要求他们拥有快速、可靠的补丁管理和事件响应流程。专门针对 PostgreSQL 配置为 Drupal 部署的威胁建模和风险评估提供了真实世界的数据点。RCE 的可能性意味着它可能被用来部署恶意软件、建立持久性或横向移动到其他系统，所有这些都属于系统安全的范畴。

批判性视角

本文基于供应商报告（Imperva、Drupal）和 CISA 公告。确切的利用方法、载荷和利用后行为并未详细说明。'权限提升和远程代码执行'的说法来自 CISA 的建议，但文章未提供独立验证或对利用链的技术分析。大多数攻击的'探测'性质表明，完整的利用能力可能尚未广泛部署，或者攻击者仍在完善其技术。CVSS 评分 6.5 相对适中，但潜在影响（RCE）是严重的；这种差异值得进一步的技术调查。
