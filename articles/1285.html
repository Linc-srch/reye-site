TrapDoor Supply Chain Attack Spreads Credential-Stealing Malware via npm, PyPI, and CratesIO

TrapDoor Supply Chain Attack Spreads Credential-Stealing Malware via npm, PyPI, and CratesIO

content-management supply-chain-attack zero-trust
启示 · Insight

For a system security researcher focused on software supply-chain security and exploitation, this campaign is a textbook example of a multi-ecosystem, cross-platform attack that weaponizes package manager trust mechanisms (postinstall hooks, build.rs, import-time execution). The use of .cursorrules and CLAUDE.md to subvert AI coding assistants is a novel attack vector that blurs the line between software supply-chain compromise and AI-assisted development pipeline poisoning. The SSH lateral movement and credential validation (AWS/GitHub API calls) indicate a mature operational capability aimed at deep infrastructure compromise, not just credential theft.

对于专注于软件供应链安全和漏洞利用的系统安全研究员而言，这次攻击是跨生态系统、跨平台攻击的教科书式案例，它武器化了包管理器的信任机制（postinstall hooks、build.rs、导入时执行）。使用 .cursorrules 和 CLAUDE.md 来颠覆 AI 编码助手是一种新颖的攻击向量，模糊了软件供应链攻陷与 AI 辅助开发流水线投毒之间的界限。SSH 横向移动和凭证验证（AWS/GitHub API 调用）表明其具备成熟的操作能力，旨在深度渗透基础设施，而不仅仅是窃取凭证。

Summary

A coordinated supply chain attack named TrapDoor has been discovered across npm, PyPI, and Crates.io, deploying over 34 malicious packages (384+ versions) targeting crypto, DeFi, Solana, and AI developers to steal credentials, crypto wallets, and cloud secrets, with novel persistence via AI coding tool configuration files.

摘要

一场名为 TrapDoor 的协同供应链攻击在 npm、PyPI 和 Crates.io 上被发现，部署了超过 34 个恶意包（384+ 版本），针对加密货币、DeFi、Solana 和 AI 开发者，窃取凭证、加密钱包和云密钥，并通过 AI 编码工具的配置文件实现新型持久化。

Deep Note

Key Points

  • Multi-ecosystem reach: 34+ malicious packages across npm, PyPI, and Crates.io (384+ versions).
  • Target profile: Developers in crypto, DeFi, Solana, and AI communities.
  • Payload delivery: npm postinstall hooks, PyPI import-time execution, Rust build.rs scripts.
  • Core payload (trap-core.js): Scans for credentials, validates AWS/GitHub tokens, performs SSH lateral movement, and establishes persistence via cron, systemd, Git hooks, shell hooks, .cursorrules, and CLAUDE.md.
  • Novel AI subversion: Hidden instructions in .cursorrules and CLAUDE.md trick AI coding assistants into exfiltrating secrets via GitHub PRs.
  • Exfiltration: Uses GitHub Gists for Rust crates; remote JavaScript payloads hosted on GitHub Pages.
  • Persistence: Multiple mechanisms including systemd, cron, Git hooks, and SSH keys.

Context & Contrast

While supply-chain attacks on package registries are well-documented (e.g., dependency confusion, typosquatting, malicious packages), TrapDoor is notable for its coordinated cross-ecosystem execution and the novel use of AI coding assistant configuration files (.cursorrules, CLAUDE.md) as both a persistence mechanism and an exfiltration channel. Previous attacks typically focused on a single ecosystem or used simpler persistence. The SSH lateral movement and credential validation against live AWS/GitHub APIs represent a higher operational security (OPSEC) maturity than typical credential-stealing campaigns.

Why It Matters

This campaign demonstrates that software supply-chain attacks are evolving beyond simple package planting into sophisticated, multi-stage operations that leverage developer tooling and AI workflows. The ability to subvert AI coding assistants to exfiltrate secrets represents a paradigm shift in attack surface. For system security, it highlights that package manager trust models are insufficient against determined adversaries, and that the integration of AI tools into development pipelines introduces new, unhardened attack vectors. The SSH lateral movement capability means a single compromised developer workstation can lead to broader infrastructure compromise.

Critical Take

The article is based on a single vendor report (Socket) and lacks independent verification of the full scope or attribution. The claim of AI subversion via .cursorrules and CLAUDE.md is novel but the practical success rate and real-world impact are unverified. The number of packages (34) and versions (384) is significant, but the actual number of downloads and infections is not provided. The campaign's operational security (using GitHub Pages and Gists) is relatively noisy and likely to be detected quickly by automated scanning. The 'TrapDoor' codename collision with a separate Android ad fraud campaign is confusing and may cause misattribution.

What to Watch

  • Detection signatures: Monitor for packages matching the naming patterns (e.g., -builder, -auditor, *-checker) across npm, PyPI, and Crates.io.
  • AI tooling hardening: Watch for updates to Cursor, Claude, and other AI coding assistants that restrict or audit .cursorrules and CLAUDE.md files.
  • Lateral movement indicators: Monitor for unexpected SSH connections from developer workstations to internal infrastructure.
  • Credential validation APIs: Track unusual API calls to AWS STS, GitHub API, or other cloud provider endpoints from build environments.
  • Registry responses: Observe how npm, PyPI, and Crates.io coordinate takedowns and whether they implement cross-ecosystem threat intelligence sharing.
深度笔记

要点

  • 多生态系统覆盖：在 npm、PyPI 和 Crates.io 上发现 34+ 个恶意包（384+ 版本）。
  • 目标画像：加密货币、DeFi、Solana 和 AI 社区的开发者。
  • 载荷投递：npm 的 postinstall hooks、PyPI 的导入时执行、Rust 的 build.rs 脚本。
  • 核心载荷 (trap-core.js)：扫描凭证，验证 AWS/GitHub 令牌，执行 SSH 横向移动，并通过 cron、systemd、Git hooks、shell hooks、.cursorrules 和 CLAUDE.md 建立持久化。
  • 新颖的 AI 颠覆：.cursorrules 和 CLAUDE.md 中的隐藏指令诱骗 AI 编码助手通过 GitHub PR 泄露秘密。
  • 数据外泄：Rust crate 使用 GitHub Gists；远程 JavaScript 载荷托管在 GitHub Pages 上。
  • 持久化：多种机制，包括 systemd、cron、Git hooks 和 SSH 密钥。

背景与对比

虽然针对包注册表的供应链攻击已有大量记录（例如依赖混淆、域名抢注、恶意包），但 TrapDoor 因其协调的跨生态系统执行以及将 AI 编码助手配置文件（.cursorrules、CLAUDE.md）作为持久化和外泄渠道的新颖用法而引人注目。以前的攻击通常专注于单一生态系统或使用更简单的持久化方式。针对实时 AWS/GitHub API 的 SSH 横向移动和凭证验证代表了比典型凭证窃取活动更高的操作安全成熟度。

重要性

这次攻击表明，软件供应链攻击正在从简单的包植入演变为复杂的多阶段操作，利用开发者工具和 AI 工作流。颠覆 AI 编码助手以泄露秘密的能力代表了攻击面的范式转变。对于系统安全而言，它凸显了包管理器的信任模型不足以应对坚定的攻击者，并且将 AI 工具集成到开发流水线中引入了新的、未经加固的攻击向量。SSH 横向移动能力意味着单个受感染的开发者工作站可能导致更广泛的基础设施沦陷。

批判性视角

该文章基于单一供应商报告（Socket），缺乏对全部范围或归因的独立验证。通过 .cursorrules 和 CLAUDE.md 进行 AI 颠覆的说法很新颖，但其实际成功率和现实世界影响尚未得到验证。包数量（34 个）和版本数量（384 个）很大，但未提供实际的下载量和感染数量。该活动的操作安全（使用 GitHub Pages 和 Gists）相对嘈杂，很可能被自动扫描快速检测到。'TrapDoor' 代号与另一个独立的 Android 广告欺诈活动冲突，可能导致归因错误。

后续关注

  • 检测特征：监控 npm、PyPI 和 Crates.io 上匹配命名模式（例如 -builder、-auditor、*-checker）的包。
  • AI 工具加固：关注 Cursor、Claude 和其他 AI 编码助手的更新，这些更新会限制或审计 .cursorrules 和 CLAUDE.md 文件。
  • 横向移动指标：监控从开发者工作站到内部基础设施的意外 SSH 连接。
  • 凭证验证 API：跟踪来自构建环境的对 AWS STS、GitHub API 或其他云提供商端点的异常 API 调用。
  • 注册表响应：观察 npm、PyPI 和 Crates.io 如何协调下架，以及它们是否实施跨生态系统威胁情报共享。
