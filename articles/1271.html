GitHub Confirms Breach, 4K Internal Repos Stolen

GitHub Confirms Breach, 4K Internal Repos Stolen

Summary

GitHub confirmed a data breach where a threat actor named TeamPCP stole approximately 4,000 internal code repositories. The breach originated from a compromised employee device via a poisoned VS Code extension. GitHub detected and contained the incident, rotated critical secrets, and is investigating. TeamPCP, known for financially motivated attacks on the open source ecosystem, offered the stolen data for sale on a dark web forum, threatening to leak it if no buyer is found. The incident highlights risks in developer tooling trust models and supply chain security.

摘要

GitHub确认发生数据泄露，攻击者TeamPCP通过被投毒的VS Code扩展入侵员工设备，窃取了约4000个内部代码仓库。GitHub已检测并控制事件，轮换了关键密钥并展开调查。TeamPCP是一个以经济为动机的威胁行为者，曾在暗网论坛出售数据，并威胁若无买家将公开泄露。此事件凸显了开发者工具信任模型和供应链安全的风险。
