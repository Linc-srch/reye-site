Malicious npm Package Stole Files From Claude AI User Directory via GitHub

Malicious npm Package Stole Files From Claude AI User Directory via GitHub

content-management supply-chain-attack zero-trust
启示 · Insight

This incident is a textbook software supply-chain attack targeting the npm ecosystem, a core concern for System Security researchers. It demonstrates a low-sophistication but effective technique: abusing the postinstall hook for arbitrary code execution and using a legitimate cloud service (GitHub) as an exfiltration channel. The attack specifically targets AI tooling data, highlighting a new and valuable attack surface — the local storage of AI assistants. For researchers focused on OS & Kernel Security and Isolation & Containment, this underscores the need for stronger sandboxing and filesystem access controls for developer tools and AI applications, as they often have broad access to user data.

此事件是针对 npm 生态系统的典型软件供应链攻击，是系统安全研究人员的核心关注点。它展示了一种低复杂度但有效的技术：滥用 postinstall 钩子执行任意代码，并使用合法云服务（GitHub）作为外泄通道。该攻击专门针对 AI 工具数据，突显了一个新的、有价值的攻击面——AI 助手的本地存储。对于专注于操作系统与内核安全以及隔离与容器化的研究人员来说，这强调了为开发者工具和 AI 应用程序实施更强的沙箱和文件系统访问控制的必要性，因为它们通常对用户数据拥有广泛的访问权限。

Summary

A malicious npm package named 'mouse5212-super-formatter' was discovered exfiltrating files from the Claude AI user data directory (/mnt/user-data) to a GitHub repository controlled by the attacker. The package used a hardcoded or environment-based GitHub token for authentication and uploaded all files during the postinstall step.

摘要

发现一个名为 'mouse5212-super-formatter' 的恶意 npm 包，它从 Claude AI 用户数据目录 (/mnt/user-data) 窃取文件，并将其上传到攻击者控制的 GitHub 仓库。该包在 postinstall 阶段使用硬编码或环境中的 GitHub 令牌进行认证并上传所有文件。

Deep Note

Key Points

  • Package name: mouse5212-super-formatter on npm.
  • Target: /mnt/user-data directory used by Anthropic's Claude AI.
  • Exfiltration method: Authenticates to GitHub (via hardcoded or env token) and uploads all files recursively to a threat-controlled GitHub repo.
  • Trigger: postinstall npm script hook.
  • Downloads: ~676 times; GitHub account created hours before first malicious version.
  • Attacker OPSEC failure: Hardcoded GitHub token leaked in the package, suggesting AI-generated malware with poor operational security.
  • Campaign name: Malware-Slop.

Context & Contrast

This attack follows a well-established pattern of malicious npm packages using postinstall scripts for data theft or cryptomining. However, it is novel in its specific targeting of AI application data (Claude's user directory) and its use of GitHub as both an authentication mechanism and an exfiltration endpoint. The sloppy OPSEC (hardcoded token) contrasts with more sophisticated supply-chain attacks that use stolen tokens or compromised maintainer accounts, suggesting a lower barrier to entry for attackers leveraging AI code generation.

Why It Matters

For System Security researchers, this matters because it highlights a growing trend: the convergence of software supply-chain attacks with AI application security. The /mnt/user-data directory represents a high-value target containing sensitive user prompts, outputs, and potentially proprietary code. It also demonstrates that even low-sophistication attacks can be effective if they target a valuable data source and exploit a widely-used distribution mechanism (npm). This reinforces the importance of runtime monitoring, filesystem access control, and the principle of least privilege for development and AI tools.

Critical Take

The article is based on a single vendor report (OX Security) and the analysis is straightforward. The 'AI-generated malware' claim is speculative, inferred from the poor OPSEC rather than confirmed. The actual impact (number of compromised systems) is unknown, as download counts do not equal installs. The attack is technically unsophisticated and easily detectable by basic security measures (e.g., monitoring outbound connections, scanning for hardcoded tokens).

What to Watch

  • Increase in similar 'sloppy' malware targeting AI tool directories (e.g., ChatGPT, Copilot local data).
  • npm registry's response and any automated malware blocking improvements.
  • Emergence of more sophisticated variants that avoid hardcoded tokens and use stolen or OAuth-based credentials.
  • Research into filesystem isolation mechanisms for AI assistants and developer tools (e.g., using containers, sandboxing, or virtualized filesystems).
深度笔记

要点

  • 包名：npm 上的 mouse5212-super-formatter。
  • 目标：Anthropic 的 Claude AI 使用的 /mnt/user-data 目录。
  • 外泄方法：通过硬编码或环境变量中的令牌认证 GitHub，并将所有文件递归上传到攻击者控制的 GitHub 仓库。
  • 触发点：npm 的 postinstall 脚本钩子。
  • 下载量：约 676 次；GitHub 账户在首个恶意版本上传前几小时创建。
  • 攻击者 OPSEC 失败：包中泄露了硬编码的 GitHub 令牌，表明这是 AI 生成的恶意软件，操作安全性差。
  • 活动名称：Malware-Slop。

背景与对比

此次攻击遵循了恶意 npm 包利用 postinstall 脚本进行数据窃取或挖矿的既定模式。但其新颖之处在于专门针对 AI 应用数据（Claude 的用户目录），并使用 GitHub 同时作为认证机制和外泄端点。其糟糕的 OPSEC（硬编码令牌）与使用窃取令牌或入侵维护者账户的更复杂供应链攻击形成对比，表明利用 AI 代码生成的攻击者入门门槛更低。

重要性

对系统安全研究人员而言，此事之所以重要，是因为它突显了一个日益增长的趋势：软件供应链攻击与 AI 应用安全的融合。/mnt/user-data 目录是一个高价值目标，包含敏感的用户提示、输出和潜在的专有代码。它还表明，即使是低复杂度的攻击，只要瞄准有价值的数据源并利用广泛使用的分发机制（npm），也能奏效。这强化了运行时监控、文件系统访问控制以及为开发工具和 AI 工具应用最小权限原则的重要性。

批判性视角

该文章基于单一供应商（OX Security）的报告，分析较为直接。关于‘AI 生成的恶意软件’的说法是推测性的，是基于糟糕的 OPSEC 推断而来，并非确凿证据。实际影响（受感染系统数量）未知，因为下载量不等于安装量。该攻击在技术上并不复杂，很容易被基本的安全措施（例如监控出站连接、扫描硬编码令牌）检测到。

后续关注

  • 针对 AI 工具目录（例如 ChatGPT、Copilot 本地数据）的类似‘粗糙’恶意软件的增加。
  • npm 注册表的响应以及任何自动恶意软件拦截的改进。
  • 出现更复杂的变种，避免使用硬编码令牌，转而使用窃取的或基于 OAuth 的凭证。
  • 针对 AI 助手和开发者工具的文件系统隔离机制的研究（例如使用容器、沙箱或虚拟化文件系统）。
