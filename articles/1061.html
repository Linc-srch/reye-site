1) Why-read

Proposes a training-free, lookup-based attention-state memory that externalizes long prefixes to eliminate inference-time prefix attention, achieving accuracy gains over ICL and RAG with lower latency and memory footprint.

2) CRGP

C — Context

LLM applications increasingly rely on long conditioning prefixes (e.g., in-context learning, RAG, agentic instructions) to guide behavior at inference. However, prefix-augmented inference suffers from prefix decay (influence fades as generation proceeds) and linear attention cost over prefix length. Existing solutions either keep the prefix in attention (compression) or internalize it via gradient-based training (context distillation, hypernetworks), but the former still incurs per-step attention cost and decay, while the latter is training-intensive and inflexible for prefix updates.

R — Related work

  • Context distillation (Snell et al., 2022; Kujanpää et al., 2024; Upadhayaya et al., 2024; Shin et al., 2025; Asawa et al., 2026; Zhang and Wang, 2026) fine-tunes model parameters to internalize prefix behavior, eliminating inference-time prefix attention but requiring expensive gradient updates.
  • Hypernetwork-based prefix internalization (Charakorn et al., 2025, 2026) amortizes per-prefix training by mapping prefixes to low-rank parameters in one forward pass, but still requires training on billions of tokens.
  • Prompt compression (Jiang et al., 2023; Li et al., 2023; Pan et al., 2024; Mu et al., 2023; Chevalier et al., 2023; Ge et al., 2023) reduces prefix length at token level (hard/soft methods), while query-agnostic KV cache compression (Kim et al., 2025; Song et al., 2026) evicts cache entries; both keep prefix in attention, so cost is reduced but not removed and decay persists.
  • Prefix caching (Kwon et al., 2023; Zheng et al., 2024; Jin et al., 2025) amortizes prefill but still incurs substantial memory consumption for long prefixes.

G — Gap

No prior method simultaneously provides prefix-length-independent decoding latency, training-free prefix construction, and no auxiliary models. Methods that keep prefix in attention preserve flexibility but pay per-query attention and suffer decay; methods that move prefix into parameters eliminate attention but require gradient updates, making them slow, memory-intensive, and ill-suited for frequent prefix updates.

P — Proposal

Proposes attention-state memory, a training-free approach that externalizes the prefix into a lightweight, lookup-based memory of precomputed attention states (attention output and softmax normalizer) between prefix and query tokens. The memory is built by running forward passes over representative queries, collecting attention outputs over the prefix, and clustering them into centroids; at inference, an incoming query retrieves the closest centroid and merges it with self-attention via the online-softmax identity, recovering the attention output without attending to the prefix. Key insight: the online-softmax identity allows lossless decomposition and recomposition of attention over sub-blocks, enabling retrieval-based prefix reuse that is training-free, avoids prefix decay, and has logarithmic lookup cost.

4) Experiments

Main results

On ManyICLBench with LLaMA-3.1-8B, attention-state memory improves accuracy over in-context learning at 1K-8K memory budgets while reducing attention latency by 1.36x at 8K. For RAG on the NBA benchmark, it surpasses full-attention RAG performance using only 20% of its memory footprint.

Ablation

The paper does not report explicit ablation results (e.g., varying number of centroids, clustering method, or query sampling strategy) with numerical comparisons; the main results focus on accuracy vs. memory budget and latency reduction.

Limitations

['The memory construction requires a set of representative queries, which may not cover all possible inference queries, potentially leading to retrieval errors for out-of-distribution queries.', 'The clustering-based retrieval introduces approximation; the paper does not analyze the impact of centroid count or clustering quality on downstream performance.', 'Evaluation is limited to LLaMA-3.1-8B and two benchmarks (ManyICLBench, NBA RAG); generalizability to other models and tasks (e.g., long-document QA, multi-turn dialogue) is not demonstrated.']

5) Why it matters

  1. Provides a training-free alternative to context distillation and hypernetworks, enabling fast prefix updates without gradient computation, which is valuable for dynamic agentic systems where prefixes change frequently.
  2. Demonstrates that externalizing prefix attention states via retrieval can match or exceed full-attention performance while reducing latency and memory, offering a practical path for scaling long-context LLM inference.
  3. Connects to your interest in efficient LLM inference and memory-augmented architectures; the online-softmax decomposition insight could inspire further work on modular attention mechanisms.
