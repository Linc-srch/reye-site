Building a safe, effective sandbox to enable Codex on Windows

Building a safe, effective sandbox to enable Codex on Windows

content-management editorial-workflow digital-publishing
Summary

OpenAI's Codex coding agent on Windows lacked a sandbox, forcing users to either approve every command or grant full access. Existing Windows isolation tools like AppContainer, Windows Sandbox, and Mandatory Integrity Control were unsuitable due to rigidity, product limitations, or security risks. The team built a custom unelevated sandbox using SIDs and write-restricted tokens to enforce file-write and network constraints without requiring administrator privileges. This approach provides granular control over where Codex can modify files, balancing safety and developer productivity on Windows.

摘要

OpenAI的Codex编码代理在Windows上缺乏沙箱，用户被迫要么批准每个命令，要么授予完全访问权限。现有的Windows隔离工具如AppContainer、Windows Sandbox和Mandatory Integrity Control因灵活性差、产品限制或安全风险而不适用。团队构建了一个自定义的非提权沙箱，利用SIDs和write-restricted tokens来强制执行文件写入和网络约束，无需管理员权限。该方法提供了对Codex可修改文件位置的精细控制，在Windows上平衡了安全性和开发效率。
