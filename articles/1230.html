Summary

Project Glasswing, a collaborative effort using Claude Mythos Preview, has discovered over 10,000 high/critical-severity vulnerabilities in critical software within a month, shifting the bottleneck from discovery to verification and patching.

摘要

Project Glasswing 利用 Claude Mythos Preview 在一个月内发现了超过 10,000 个高危/严重漏洞，将瓶颈从发现转移到了验证和修补。

Deep Note

Key Points

  • Project Glasswing partners (including Cloudflare, Mozilla, Palo Alto Networks, Microsoft, Oracle) have collectively found >10,000 high/critical vulnerabilities in one month.
  • Cloudflare found 2,000 bugs (400 high/critical) with false positive rate better than human testers.
  • Mozilla found 271 vulnerabilities in Firefox 150 with Mythos Preview (10x more than with Claude Opus 4.6).
  • UK AI Security Institute reports Mythos Preview is first model to solve both cyber ranges end-to-end.
  • Open-source scanning found 6,202 estimated high/critical vulnerabilities across 1,000+ projects; 1,752 assessed with 90.6% true-positive rate and 62.4% confirmed high/critical.
  • Example: wolfSSL vulnerability (CVE-2026-5194) allowing certificate forgery, now patched.
  • Bottleneck shifted from discovery to human triage, verification, and patch deployment.
  • Palo Alto Networks released 5x more patches than usual; Microsoft reports patch volumes will continue trending larger.

Context & Contrast

Prior to this, AI-assisted vulnerability discovery was limited to small-scale demonstrations or specific benchmarks. This represents the first large-scale, real-world deployment where an AI model (Mythos Preview) has been systematically applied to thousands of critical software projects with verified results. The 10x improvement over previous state-of-the-art (Claude Opus 4.6) and the shift from discovery bottleneck to patch bottleneck mark a fundamental change in the cybersecurity landscape.

Why It Matters

This demonstrates that AI models have crossed a threshold where they can autonomously discover exploitable vulnerabilities in production software at scale. The 90.6% true-positive rate on open-source code is particularly significant because it shows reliability. For the field, this means: (1) vulnerability discovery is no longer a human-limited task; (2) the security industry must fundamentally re-engineer its patch management and disclosure processes; (3) AI models themselves become both the primary threat vector and primary defense tool simultaneously.

Critical Take

The reported numbers are impressive but should be interpreted with caution: (1) The 10,000+ figure includes all partner findings, not just independently verified ones; (2) The open-source true-positive rate (90.6%) is based on only 1,752 of 23,019 total findings; (3) The severity classification relies on Mythos Preview's own estimates for many findings; (4) The dashboard shows a steep drop-off at each phase, indicating significant human effort still required; (5) The wolfSSL example, while real, is a single data point; (6) The long-term impact on software security depends on patch adoption rates, which remain uncertain.

What to Watch

  1. Patch adoption rates for the disclosed vulnerabilities, especially in open-source projects. 2. Whether Mythos Preview's successors maintain or improve the true-positive rate. 3. Emergence of automated patch generation tools to address the new bottleneck. 4. Regulatory responses to AI-discovered vulnerabilities and disclosure timelines. 5. Adversarial use of similar models for zero-day discovery. 6. Impact on bug bounty programs and vulnerability markets. 7. Evolution of the open-source vulnerability dashboard over time.
深度笔记

要点

  • Project Glasswing 合作伙伴（包括 Cloudflare、Mozilla、Palo Alto Networks、Microsoft、Oracle）在一个月内共同发现了超过 10,000 个高危/严重漏洞。
  • Cloudflare 发现了 2,000 个漏洞（400 个高危/严重），误报率优于人类测试人员。
  • Mozilla 使用 Mythos Preview 在 Firefox 150 中发现了 271 个漏洞（是使用 Claude Opus 4.6 的 10 倍）。
  • 英国 AI 安全研究所报告称，Mythos Preview 是首个端到端解决其两个网络靶场的模型。
  • 开源扫描在 1,000 多个项目中发现了 6,202 个估计的高危/严重漏洞；1,752 个经过评估，真阳性率为 90.6%，62.4% 被确认为高危/严重。
  • 示例：wolfSSL 漏洞（CVE-2026-5194）允许证书伪造，现已修补。
  • 瓶颈已从发现转移到人工分类、验证和补丁部署。
  • Palo Alto Networks 发布的补丁数量是平时的 5 倍；Microsoft 报告补丁数量将继续增加。

背景与对比

在此之前，AI 辅助漏洞发现仅限于小规模演示或特定基准测试。这是首次大规模、真实世界的部署，AI 模型（Mythos Preview）被系统地应用于数千个关键软件项目并取得了验证结果。相比之前的最先进技术（Claude Opus 4.6）提升了 10 倍，以及从发现瓶颈到补丁瓶颈的转变，标志着网络安全格局的根本性变化。

重要性

这表明 AI 模型已经跨越了一个门槛，能够大规模自主发现生产软件中的可利用漏洞。开源代码上 90.6% 的真阳性率尤其重要，因为它显示了可靠性。对于该领域而言，这意味着：(1) 漏洞发现不再是受限于人类的任务；(2) 安全行业必须从根本上重新设计其补丁管理和披露流程；(3) AI 模型本身同时成为主要的威胁向量和主要的防御工具。

批判性视角

报告的数字令人印象深刻，但应谨慎解读：(1) 超过 10,000 的数字包括所有合作伙伴的发现，而不仅仅是独立验证的；(2) 开源真阳性率 (90.6%) 仅基于 23,019 个总发现中的 1,752 个；(3) 严重性分类依赖于 Mythos Preview 对许多发现的自身估计；(4) 仪表板显示每个阶段都有急剧下降，表明仍然需要大量人工努力；(5) wolfSSL 示例虽然是真实的，但只是一个数据点；(6) 对软件安全的长期影响取决于补丁采用率，这仍然不确定。
