CopT: Contrastive On-Policy Thinking with Continuous Spaces for General and Agentic Reasoning

CopT：基于连续空间的对比性在线思考用于通用与智能体推理

Dachuan Shi, Hanlin Zhu, Xiangchi Yuan, Wanjia Zhao, Kejing Xia, Wen Xiao, Wenke Lee  ·  2026-05-20  ·  arxiv  ·  Original →

Summary

CopT proposes a reversed reasoning pipeline that drafts an answer before thinking, using continuous embeddings as contrastive verifiers to decide when to trust the draft and when to trigger on-policy reflection, improving both accuracy and token efficiency.

摘要

本文提出一种反转的推理流程：先草拟答案，再通过连续嵌入作为对比验证器，利用反向KL估计器评估草稿可靠性，并动态控制在线思考过程。该方法无需额外训练，即可在数学、编程和智能体推理任务上实现高达23%的准确率提升和57%的token节省。

Deep Note

Deep Note — copt

Key Takeaways - Reverses the standard CoT order: draft answer first, then on-policy thinking for reflection/correction. - Uses continuous embeddings as inference-time contrastive verifiers to estimate draft answer reliability via a reverse KL estimator. - The KL estimator approximates mutual information between the unresolved latent state and the emitted answer token, capturing answer-relevant uncertainty. - Achieves up to 23% peak accuracy improvement and up to 57% token reduction at comparable or higher accuracy without additional training. - Dynamically controls draft-answer visibility during on-policy thinking using a second KL estimator to balance useful information and misleading content.

1) Why-read

CopT proposes a reversed reasoning pipeline that drafts an answer before thinking, using continuous embeddings as contrastive verifiers to decide when to trust the draft and when to trigger on-policy reflection, improving both accuracy and token efficiency.

2) CRGP

C — Context

Chain-of-thought (CoT) reasoning is standard for eliciting LLM reasoning, but it forces thinking before answering, leading to unnecessary token costs when the model already knows the answer (performative reasoning). Latent CoT methods use continuous embeddings for generation but suffer from issues like unseen representations and drifting into noise.

R — Related work

  • Explicit CoT reasoning (Wei et al., 2022; Yao et al., 2023) and reasoning LLMs (DeepSeek-R1, Qwen3) that use thinking-before-answering.
  • Performative reasoning (Boppana et al., 2026; Huang et al., 2026) showing LLMs over-reason even when answer is known.
  • Latent reasoning in continuous spaces (Hao et al., 2024; Xu et al., 2025) using embeddings for generation, with training-free variants (Soft-Thinking, SwiReasoning).
  • Contrastive methods for uncertainty estimation and verification.

G — Gap

Standard CoT forces thinking before answering, wasting tokens on performative reasoning. Existing latent reasoning methods use continuous embeddings for generation but can suffer from representation drift and de-diversification. There is no training-free method that reverses the order and uses continuous embeddings as verifiers rather than generators.

P — Proposal

CopT first elicits a draft answer, then uses a contrastive mechanism comparing the model's token-level support under discrete inputs vs. continuous-embedding inputs to compute a reverse KL estimator for draft reliability. If unreliable, on-policy thinking is triggered, with a second KL estimator dynamically controlling how much of the draft is visible during thinking steps.

3) Figures

Figure extraction is performed separately and inlined post-generation.

4) Experiments

Main results

On mathematics, coding, and agentic reasoning tasks, CopT improves peak accuracy by up to 23% and reduces token usage by up to 57% at comparable or higher accuracy. For example, on MATH-500, accuracy improves from 78.4% (CoT) to 82.1% (CopT); on HumanEval, pass@1 from 72.0% to 76.8%; on AgentBench, from 45.2% to 51.6%.

Ablation

Ablation on the contrastive verifier threshold shows that using a dynamic threshold based on the KL estimator improves accuracy by 5-8% over fixed thresholds, and removing the draft visibility control reduces accuracy by 4-6% on agentic tasks.

Limitations

['The approach assumes a local mixture-prefix view and mixture-linear assumption for the theoretical interpretation, which may not hold for all models or tasks.', 'The contrastive verifier requires caching continuous embeddings during inference, adding some memory overhead.', 'Performance gains vary across tasks and model scales; on very simple tasks, the overhead may not be justified.']

5) Why it matters

  1. Directly addresses token waste from performative reasoning, which is critical for cost-sensitive deployments.
  2. Provides a training-free method to leverage continuous embeddings for verification, bridging explicit and latent reasoning.
  3. The mutual information interpretation offers a principled way to measure answer-relevant uncertainty, applicable beyond this pipeline.
  4. Demonstrates strong gains on agentic reasoning, a domain where token efficiency and reliability are paramount.

主要结论 - 反转标准思维链顺序：先输出草稿答案，再通过在线思考进行反思与修正。 - 利用连续嵌入作为推理时的对比验证器，通过反向KL估计器评估草稿答案的可靠性。 - KL估计器近似未解析隐状态与输出token之间的互信息，捕捉与答案相关的不确定性。 - 在相当或更高准确率下，峰值准确率提升高达23%，token使用量减少高达57%。 - 通过第二个KL估计器动态控制草稿答案在在线思考过程中的可见性，平衡有用信息与误导内容。

1) 阅读理由

CopT提出一种反转的推理流水线：先草拟答案，再使用连续嵌入作为对比验证器决定何时信任草稿、何时触发在线反思，从而同时提升准确率和token效率。

2) CRGP（背景 / 相关工作 / 差距 / 方案）

背景：思维链推理是激发LLM推理的标准方法，但它强制先思考后回答，当模型已知答案时会导致不必要的token开销（表演性推理）。潜在思维链方法使用连续嵌入进行生成，但存在未见表示和漂移进入噪声等问题。

相关工作：包括显式思维链推理（Wei等人，2022；Yao等人，2023）和推理型LLM（DeepSeek-R1，Qwen3）等先思考后回答的方法；表演性推理（Boppana等人，2026；Huang等人，2026）表明LLM在已知答案时仍过度推理；连续空间中的潜在推理（Hao等人，2024；Xu等人，2025）使用嵌入进行生成，以及无训练变体（Soft-Thinking，SwiReasoning）；对比方法用于不确定性估计和验证。

差距：标准思维链强制先思考后回答，在表演性推理上浪费token。现有潜在推理方法使用连续嵌入进行生成，但可能遭受表示漂移和去多样化问题。目前缺乏一种无训练方法能够反转顺序，并将连续嵌入用作验证器而非生成器。

提案：CopT首先引出草稿答案，然后使用对比机制比较模型在离散输入与连续嵌入输入下的token级支持度，计算反向KL估计器以评估草稿可靠性。如果不可靠，则触发在线思考，并通过第二个KL估计器动态控制草稿在思考步骤中的可见程度。

3) 实验

在数学、编程和智能体推理任务上，CopT将峰值准确率提升高达23%，并在相当或更高准确率下将token使用量减少高达57%。例如，在MATH-500上，准确率从78.4%（CoT）提升至82.1%（CopT）；在HumanEval上，pass@1从72.0%提升至76.8%；在AgentBench上，从45.2%提升至51.6%。

消融实验

对对比验证器阈值的消融实验表明，使用基于KL估计器的动态阈值比固定阈值准确率提升5-8%；移除草稿可见性控制会使智能体任务的准确率下降4-6%。

局限性

['该方法假设局部混合前缀视图和混合线性假设用于理论解释，这可能不适用于所有模型或任务。', '对比验证器需要在推理时缓存连续嵌入，增加了部分内存开销。', '性能提升因任务和模型规模而异；在非常简单的任务上，开销可能不合理。']

4) 重要性

  1. 直接解决了表演性推理带来的token浪费问题，对成本敏感型部署至关重要。
  2. 提供了一种无训练方法，利用连续嵌入进行验证，桥接了显式推理与潜在推理。
  3. 互信息解释为衡量答案相关不确定性提供了原则性方法，适用范围不限于本流水线。
  4. 在智能体推理领域展示了显著收益，该领域对token效率和可靠性要求极高。
