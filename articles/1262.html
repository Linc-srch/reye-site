Claude Mythos AI Finds 10,000 High-Severity Flaws in Widely Used Software

Claude Mythos AI Finds 10,000 High-Severity Flaws in Widely Used Software

content-management rlhf supply-chain-attack
启示 · Insight

For AI Security: This is a landmark empirical validation of frontier LLMs (Claude Mythos) as autonomous vulnerability discovery engines. It directly supports the researcher's core objective of automated red-teaming and evaluation, demonstrating that models can now find real-world, high-severity flaws at scale. The finding that the model excels at turning vulnerabilities into end-to-end attack chains is critical for developing automated, mathematically grounded frameworks for evaluating agentic and autonomous systems. For System Security: This represents a paradigm shift in vulnerability research, moving from manual or fuzzer-driven discovery to AI-driven, source-code analysis at an unprecedented scale. The 10,000+ figure, while including false positives, shows the raw power of LLMs for static analysis, potentially outpacing traditional fuzzing and manual auditing for certain classes of bugs. The focus on 'systemically important' software (e.g., WolfSSL) aligns perfectly with the domain's core objective of securing the computing stack.

对 AI 安全而言：这是前沿大语言模型（Claude Mythos）作为自主漏洞发现引擎的里程碑式实证验证。它直接支持了研究者关于自动化红队测试和评估的核心目标，证明了模型现在能够大规模地发现真实世界的高严重性漏洞。该模型擅长将漏洞转化为端到端攻击链的发现，对于开发评估自主智能体系统的自动化、数学基础框架至关重要。对系统安全而言：这代表了漏洞研究的范式转变，从手动或基于模糊测试的发现，转向以前所未有规模进行的、由 AI 驱动的源代码分析。超过 10,000 的数字（尽管包含误报）展示了 LLM 在静态分析方面的原始能力，在某些类型的漏洞上可能超越传统的模糊测试和人工审计。对“系统重要性”软件（如 WolfSSL）的关注与该领域保护计算堆栈的核心目标完全一致。

Summary

Anthropic's Project Glasswing, using the Claude Mythos Preview model, has uncovered over 10,000 high/critical severity vulnerabilities in widely-used open-source software within a month, with 1,094 confirmed high/critical flaws and 97 already patched upstream.

摘要

Anthropic 的 Project Glasswing 利用 Claude Mythos Preview 模型，在一个月内发现了广泛使用的开源软件中超过 10,000 个高危/严重漏洞，其中 1,094 个被确认为高危/严重漏洞，97 个已被上游修复。

Deep Note

Key Points

  • Anthropic's Project Glasswing has identified over 10,000 high/critical severity vulnerabilities in widely-used software.
  • 6,202 of these are high/critical severity flaws impacting over 1,000 open-source projects.
  • 1,726 are validated true positives, with 1,094 assessed as high/critical severity.
  • A critical flaw in WolfSSL (CVE-2026-5194, CVSS 9.1) was found, allowing certificate forgery.
  • 97 findings have been patched upstream, and 88 advisories issued.
  • The model is described as 'substantially better than prior models at finding vulnerability candidates' and 'adept at analyzing source code with a security mindset'.
  • Mythos Preview can also turn vulnerabilities into end-to-end attack chains.
  • A partner bank used the model to detect and prevent a fraudulent $1.5 million wire transfer.
  • Anthropic launched a Cyber Verification Program for legitimate security research without guardrails.
  • The model is not publicly released due to misuse concerns.

Context & Contrast

This is a significant leap from prior AI-assisted vulnerability discovery. Previous work (e.g., fuzzing with ML guidance, or LLMs for patch analysis) was more narrow. This demonstrates a frontier model performing autonomous, end-to-end discovery at scale on real-world, critical software. It contrasts with the 'AI hype' by providing concrete numbers (10k flaws, 1k+ projects, 97 patches). It also contrasts with the defensive focus of most AI safety research by showing offensive capability used for defense. The scale (10k in a month) dwarfs typical manual or fuzzer-driven discovery campaigns.

Why It Matters

For AI Security: It provides a powerful new tool for automated red-teaming and evaluation, moving from theoretical jailbreaks to practical, high-impact vulnerability discovery. It validates the concept of using frontier models for security auditing, which is a core research area. For System Security: It signals a fundamental shift in the vulnerability landscape. The 'relative ease of finding vulnerabilities compared with the difficulty of fixing them' is a new, critical challenge. It forces a re-evaluation of patch cycles and defensive postures, as the attacker's discovery capability is accelerating. The finding of a critical flaw in a widely-used TLS library (WolfSSL) has immediate, practical implications for internet security.

Critical Take

The 10,000 figure is raw candidates; only 1,726 were validated true positives, and 1,094 high/critical. The false positive rate is high (~83%). The model is not publicly available, and its capabilities are only demonstrated by Anthropic and a small set of partners. The claim of 'turning vulnerabilities into end-to-end attack chains' needs independent verification. The effectiveness against complex, logic-based bugs (vs. memory corruption) is not detailed. The long-term impact on patch management is a challenge, not a solved problem.

What to Watch

  • Independent validation of Mythos Preview's capabilities by third-party researchers (e.g., XBOW's detailed reports).
  • The number of CVEs assigned and patches released stemming from this project over the next 6-12 months.
  • Public release or wider availability of similar models (e.g., GPT-5.5-Cyber) and their comparative performance.
  • The impact on patch cycles: will major vendors (like Oracle) accelerate to monthly or even weekly cycles?
  • Emergence of new defensive techniques specifically designed to counter AI-driven vulnerability discovery.
  • The evolution of the 'Cyber Verification Program' and how it balances utility and misuse risk.

背景与对比

这与之前 AI 辅助的漏洞发现相比是一次重大飞跃。之前的工作（例如，带有 ML 指导的模糊测试，或用于补丁分析的 LLM）范围更窄。这展示了一个前沿模型在现实世界的关键软件上自主、大规模地进行端到端发现。它通过提供具体数字（10,000 个漏洞，1,000+ 个项目，97 个补丁）与“AI 炒作”形成对比。它通过展示用于防御的进攻能力，也与大多数 AI 安全研究的防御性重点形成对比。其规模（一个月内 10,000 个）使典型的手动或基于模糊测试的发现活动相形见绌。

重要性

对 AI 安全而言：它为自动化红队测试和评估提供了一个强大的新工具，从理论上的越狱转向了实际的、高影响力的漏洞发现。它验证了使用前沿模型进行安全审计的概念，这是一个核心研究领域。对系统安全而言：它标志着漏洞格局的根本性转变。“发现漏洞的相对容易与修复漏洞的困难相比”是一个新的关键挑战。它迫使重新评估补丁周期和防御态势，因为攻击者的发现能力正在加速。在广泛使用的 TLS 库（WolfSSL）中发现关键漏洞，对互联网安全具有直接、实际的影响。

批判性视角

10,000 的数字是原始候选数；只有 1,726 个被验证为真阳性，1,094 个为高危/严重。误报率很高（约 83%）。该模型尚未公开，其能力仅由 Anthropic 和一小部分合作伙伴展示。关于“将漏洞转化为端到端攻击链”的说法需要独立验证。针对复杂的、基于逻辑的漏洞（相对于内存损坏）的有效性尚未详细说明。对补丁管理的长期影响是一个挑战，而不是一个已解决的问题。
