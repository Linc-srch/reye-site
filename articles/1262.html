Claude Mythos AI Finds 10,000 High-Severity Flaws in Widely Used Software

Claude Mythos AI Finds 10,000 High-Severity Flaws in Widely Used Software

content-management rlhf supply-chain-attack
启示 · Insight

For AI Security: This demonstrates a paradigm shift from manual red-teaming to automated, large-scale vulnerability discovery by frontier LLMs. The model's ability to autonomously identify flaws and even construct end-to-end attack chains directly validates the core research objective of building automated evaluation frameworks. The 1,726 true positives from 10,000 candidates highlight the need for better automated validation pipelines (e.g., Claw-Eval) to filter noise. For System Security: This is a watershed moment for vulnerability research. The scale (10k flaws in a month) dwarfs traditional human-led efforts. The finding that fixing is harder than finding underscores a new bottleneck: patch management and supply-chain hygiene. The model's success in analyzing source code with a 'security mindset' suggests that AI-assisted vulnerability discovery will become the new baseline, forcing defenders to accelerate patch cycles (as Oracle has done).

对于 AI 安全：这展示了从手动红队测试到由前沿大语言模型驱动的自动化、大规模漏洞发现的范式转变。该模型自主识别漏洞甚至构建端到端攻击链的能力，直接验证了构建自动化评估框架的核心研究目标。从 10,000 个候选漏洞中确认 1,726 个真阳性，凸显了对更好自动化验证流水线（例如 Claw-Eval）以过滤噪音的需求。对于系统安全：这是漏洞研究的一个分水岭时刻。其规模（一个月内发现 10,000 个漏洞）使传统人工主导的努力相形见绌。修复比发现更难这一发现突显了一个新的瓶颈：补丁管理和软件供应链卫生。该模型以‘安全思维’分析源代码的成功表明，AI 辅助的漏洞发现将成为新基线，迫使防御者加速补丁周期（正如 Oracle 所做的那样）。

Summary

Anthropic's Project Glasswing, using the Claude Mythos Preview model, has uncovered over 10,000 high/critical severity vulnerabilities in widely-used software, with 1,726 confirmed true positives and 97 already patched upstream.

摘要

Anthropic 的 Project Glasswing 利用 Claude Mythos Preview 模型，在广泛使用的软件中发现了超过 10,000 个高危/严重漏洞，其中 1,726 个被确认为真阳性，97 个已被上游修复。

Deep Note

Key Points

  • Anthropic's Project Glasswing discovered over 10,000 high/critical severity vulnerabilities in systemically important software.
  • 6,202 flaws impact over 1,000 open-source projects; 1,726 are confirmed true positives; 1,094 are high/critical.
  • One critical flaw found: WolfSSL CVE-2026-5194 (CVSS 9.1) allowing certificate forgery.
  • 97 findings have been patched upstream; 88 advisories issued.
  • Claude Mythos Preview is described as 'substantially better than prior models' at finding vulnerability candidates and adept at analyzing source code with a security mindset.
  • The model can also turn vulnerabilities into end-to-end attack chains.
  • Anthropic urges shorter patch cycles; Oracle has already shifted to monthly patches.
  • Anthropic launched a Cyber Verification Program for legitimate security use without guardrails, similar to OpenAI's Daybreak.

Context & Contrast

Prior AI-assisted vulnerability discovery (e.g., fuzzing, static analysis tools) was largely semi-automated and required significant human effort to triage. This marks a leap to fully autonomous, large-scale discovery by a frontier LLM. The scale (10k in a month) and the model's ability to construct attack chains from raw findings are unprecedented. It contrasts with earlier red-teaming efforts that focused on single models or small codebases; here the target is the global software supply chain.

Why It Matters

For AI Security: It validates that frontier LLMs can serve as powerful, scalable red-teaming tools for real-world software, not just LLM safety. This opens a new research axis: using LLMs to secure the AI supply chain itself. For System Security: It signals a fundamental shift in vulnerability research economics. The bottleneck moves from discovery to remediation. Defenders must adopt faster patch cycles and automated validation. The model's success also raises the bar for exploit prevention, as attackers will soon have similar capabilities.

Critical Take

The 1,726 true positives out of 10,000 candidates (17.3% precision) indicates significant noise; the model may be overcounting or flagging low-impact issues. The claim of 'high/critical severity' for 1,094 flaws needs independent verification. The WolfSSL CVE is a single data point; broader impact across the 1,000 projects is unverified. Anthropic's framing as a 'defensive effort' is self-serving; the same model could be used offensively. The Cyber Verification Program's guardrails and misuse potential remain unclear.

What to Watch

  • Independent validation of the 1,094 high/critical severity claims by third-party researchers.
  • Adoption of monthly patch cycles by other major vendors (beyond Oracle).
  • Public release or leakage of Mythos-class models and subsequent misuse.
  • Development of automated patch generation tools to match the discovery rate.
  • Emergence of similar capabilities in open-source or adversarial models (e.g., GPT-5.5-Cyber).
  • Impact on software supply-chain security practices (SBOM, dependency scanning).

背景与对比

之前的 AI 辅助漏洞发现（例如模糊测试、静态分析工具）大多是半自动化的，需要大量人工进行分类。这标志着向由前沿大语言模型驱动的全自主、大规模发现的飞跃。其规模（一个月内 10,000 个）以及模型从原始发现构建攻击链的能力是前所未有的。这与早期专注于单个模型或小型代码库的红队测试工作形成对比；这里的目标是全球软件供应链。

重要性

对于 AI 安全：它验证了前沿大语言模型可以作为强大、可扩展的红队测试工具用于现实世界的软件，而不仅仅是 LLM 安全。这开辟了一个新的研究轴：使用 LLM 来保护 AI 供应链本身。对于系统安全：它标志着漏洞研究经济学的根本性转变。瓶颈从发现转移到修复。防御者必须采用更快的补丁周期和自动化验证。该模型的成功也提高了漏洞利用预防的门槛，因为攻击者很快将拥有类似的能力。

批判性视角

从 10,000 个候选漏洞中确认 1,726 个真阳性（17.3% 的精确率）表明存在显著噪音；该模型可能过度计数或标记了低影响问题。声称 1,094 个漏洞为‘高危/严重’需要独立验证。WolfSSL CVE 是单一数据点；对 1,000 个项目的更广泛影响尚未得到验证。Anthropic 将其描述为‘防御性努力’是利己的；同样的模型可能被用于攻击。网络验证计划的护栏和滥用潜力仍不清楚。
