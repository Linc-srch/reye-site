Google API Keys Remain Active After Deletion

Google API Keys Remain Active After Deletion

content-management supply-chain-attack zero-trust
Summary

This paper reveals a critical security flaw in Google Cloud Platform where API keys remain active even after being deleted by users. The researchers demonstrate that deleted keys can still be used for authentication, posing a significant risk of unauthorized access and data breaches. The issue stems from caching mechanisms and eventual consistency models in distributed systems. The findings highlight the need for immediate revocation enforcement and improved key lifecycle management in cloud services.

摘要

本文揭示了Google Cloud Platform中的一个关键安全漏洞：用户删除API密钥后，这些密钥仍然保持激活状态。研究人员证明，已删除的密钥仍可用于身份验证，构成未授权访问和数据泄露的重大风险。该问题源于分布式系统中的缓存机制和最终一致性模型。研究结果强调了在云服务中实施即时撤销和改进密钥生命周期管理的必要性。
