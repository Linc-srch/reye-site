1) Why-read

A-ProS proposes a hybrid multi-model feedback framework that separates solution generation from specialized debugging, achieving over 2x greater gains than baseline agent loops on competitive programming problems.

2) CRGP

C — Context

Large Language Models (LLMs) have shown strong potential for automated code generation but typically operate as static predictors with limited ability to iteratively refine solutions. Competitive programming provides a rigorous testbed for evaluating end-to-end algorithmic reasoning and precise implementation under strict constraints.

R — Related work

  • Static benchmarks like Codeforces and APPS treat code generation as a single-attempt prediction task
  • Prior work improves single-attempt accuracy through better prompting or larger models (e.g., Codex, AlphaCode)
  • Few studies explore multi-model collaboration where specialized agents exchange structured feedback for iterative refinement
  • Agentic LLMs have been applied to test repair, bug localization, and code refactoring but not systematically to competitive programming

G — Gap

Most prior approaches treat competitive programming as a static benchmark, neglecting iterative feedback, error-aware refinement, and multi-model specialization. The field lacks systematic evidence comparing the contributions of code-specialized and reasoning-focused critics within collaborative debugging workflows.

P — Proposal

A-ProS is an autonomous AI agent that combines ChatGPT-based generators (GPT-4 and GPT-5) with three debugging critics (Codestral-2508, Llama-3.3-70B, DeepSeek-R1) under a 2x3 factorial design. The framework separates solution generation from specialized debugging, using stateful refinement with persistent conversational context to iteratively improve solutions over three rounds.

3) Figures

Figure extraction is performed separately and inlined post-generation.

4) Experiments

Main results

Evaluated on 367 problems (167 ICPC World Finals 2011-2024, 200 Codeforces rated 1200-1800). GPT-5 workflows improved from 39 initial accepted solutions to 85-90 after three refinement rounds; GPT-4 improved from 15 to 31-38. A-ProS achieved over 2x greater gains compared to baseline agent loops.

Ablation

Stateful refinement outperformed stateless approaches by 8.5-10.6 percentage points in Itr 3 acceptance rates and reduced repeated failures by up to 3.5x (bootstrap 95% CIs: [0.00, +0.15] for GPT-5, [0.00, +0.11] for GPT-4; GPT-5 effect McNemar p≈0.063).

Limitations

Evaluation is scoped to competitive programming, which may not generalize to broader software engineering tasks. The framework relies on authenticated browser automation for live judge submission, limiting reproducibility for ICPC/Gym problems. The ablation study uses a relatively small sample (n=47) and the GPT-5 stateful effect is only marginally significant.

5) Why it matters

  1. Demonstrates that persistent multi-model feedback loops provide qualitatively distinct benefits over stateless refinement, relevant to your work on agentic systems and iterative reasoning
  2. Provides a rigorous factorial design comparing code-specialized vs. reasoning-focused critics, informing model selection for debugging pipelines
  3. Shows stateful context reduces error repetition by up to 3.5x, a key insight for building reliable autonomous programming assistants
  4. The 2x improvement over baseline loops highlights the importance of coordinated multi-model collaboration over single-model refinement
